# 2026年6月11日分：.NET 11 Preview 5 と GitHub Copilot の重要更新

## 先に結論

今日は、**.NET 11 Preview 5 の実装改善** と、**GitHub Copilot の重要な運用変更** を押さえておくのがよさそうです。  
.NET 側は開発体験の底上げ、GitHub 側は既存ユーザーの使い方とコスト管理に直結する更新でした。

## Developer Blogs（.NET 目線で読む）

### .NET 11 Preview 5 は、日々の開発を軽くする更新が多い

- [.NET 11 Preview 5 is now available!](https://devblogs.microsoft.com/dotnet/dotnet-11-preview-5/)

今回の Preview 5 は、単発の大技よりも、**普段の実装で効く改善** が目立ちます。

特に見ておきたいのは次のあたりです。

1. `System.Text.Json` が JSON Lines に対応した  
2. LINQ に full outer join が入った  
3. SDK のビルド時に脆弱性 / EOL チェックができるようになった  
4. `dotnet new` に MCP Server テンプレートが追加された  
5. Blazor SSR や EF Core まわりも着実に前進した

要するに、.NET 11 は「目新しい機能を足す」だけではなく、**ビルド・ライブラリ・Web・データアクセスの摩擦を減らす方向** に進んでいます。

> Preview は派手さより、次の安定版で効いてくる“地味だけど強い改善”を拾うのが大事です。

## GitHub Changelog（過去1日で目立った更新）

### Copilot CLI に `/security-review` が追加された

- [Dedicated security review command now available in Copilot CLI](https://github.blog/changelog/2026-06-10-dedicated-security-review-command-now-available-in-copilot-cli/)

GitHub Copilot CLI で、**`/security-review` を直接叩ける** ようになりました。  
ローカルの変更差分を見て、ターミナル内でセキュリティ観点の指摘を返してくれます。

既存ユーザーに効くのは、これが**作業フローの中にそのまま入る** ことです。  
コードスキャンや Dependabot を置き換えるものではありませんが、コミット前に軽く当てるレビューとして使いやすいです。

### Enterprise の cost center 上限が 500 に拡大した

- [Enterprises can now create up to 500 cost centers](https://github.blog/changelog/2026-06-10-enterprises-can-now-create-up-to-500-cost-centers/)

enterprise で作れる cost center の上限が **250 から 500 に倍増** しました。  
大きめの組織では、部署や製品ごとの費用配賦をかなり細かく切れるようになります。

こちらは機能の派手さより、**管理の粒度を上げたい既存ユーザーに効く更新** です。

## まとめ

- **.NET 11 Preview 5** は、開発の手間を減らす改善が中心
- **GitHub Copilot / GitHub Enterprise** は、運用を少し前に進める更新が来た
- 今日の 2 本は、どちらも **実際の使い方を変える更新** でした
