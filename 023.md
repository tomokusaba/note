# 2026年6月6日分：.NET 開発者向けの最新ブログと GitHub Changelog

## 先に結論

執筆時点では、[.NET Blog](https://devblogs.microsoft.com/dotnet/?WT.mc_id=DT-MVP-5004827) に直近 1 日の新着はありませんでした。  
その代わり、.NET 開発者にそのまま効く **Microsoft Developer Blogs の最新記事** が 2 本あり、GitHub Changelog では **Copilot と課金・利用量 API** の更新が目立っています。

## Developer Blogs（.NET 目線で読む）

### エージェントは「最新ドキュメント」を見て動くべき、という話

- [Improve your agentic developer tools by grounding in Microsoft Learn](https://devblogs.microsoft.com/blog/improve-your-agentic-developer-tools-by-grounding-in-microsoft-learn?WT.mc_id=DT-MVP-5004827)

この投稿の要点は、**AI エージェントの出力品質はモデル性能だけでなく、どの情報源を見ているかで大きく変わる** ということです。  
Microsoft Learn MCP Server を使うと、エージェントが Microsoft Learn の最新情報を直接参照できるので、古い API や deprecated な手順を踏みにくくなります。

例として、Azure AI Foundry を作るスクリプト生成で、Learn を見ない場合は古い `az ml` へ流れやすい一方、Learn を見ている場合は `az cognitiveservices` という現行の導線を選べています。  
`.NET` 自体のコードだけでなく、**Azure や Bicep、MCP、Copilot CLI をまたぐ作業** で特に効く内容です。

### .NET 開発で Copilot をどう使い分けるかが整理されている

- [Doing More with GitHub Copilot as a .NET Developer](https://devblogs.microsoft.com/dotnet/doing-more-with-github-copilot/?WT.mc_id=DT-MVP-5004827)

こちらは、.NET 開発者が Copilot を **「どの画面で、何を任せるか」** で考えるための記事です。  
補完だけでなく、Visual Studio の chat、VS Code、Copilot CLI、cloud coding agent を使い分けると、単なるコード生成よりも実務に近い使い方になります。

記事では、次の切り分けがわかりやすいです。

- **chat**: 理解、比較、計画、説明
- **agentic workflows**: 修正、検証、再実行、差分レビュー

`.NET` の現場でよくある「既存サービスの把握」「テスト追加」「失敗したビルドの切り分け」「複数ファイルにまたがる変更」にそのまま当てはまる内容でした。

## GitHub Changelog（既存ユーザーへの影響が大きいもの）

### Actions の失敗を Copilot にそのまま直させられる

- [Fix with Copilot for failing Actions now in Pro, Pro+, and Max](https://github.blog/changelog/2026-06-04-fix-with-copilot-for-failing-actions-now-in-pro-pro-and-max)

GitHub Actions のジョブ失敗画面から、**Fix with Copilot** で Copilot cloud agent に修正を依頼できます。  
失敗原因の調査、修正ブランチへの push、レビュー依頼までを一気通貫でやるので、既存ユーザーほど恩恵が大きい更新です。

### Copilot cloud agent を REST API から起動・追跡できる

- [Agent tasks REST API now available for Copilot Pro, Pro+, and Max](https://github.blog/changelog/2026-06-04-agent-tasks-rest-api-now-available-for-copilot-pro-pro-and-max)

Copilot cloud agent の task を **API で起動・追跡** できるようになりました。  
リポジトリ横断のリファクタや新規 repo の初期化、定期リリース準備のような **自動化に組み込みやすい** のが強みです。

### 課金と利用量の API が実運用向けに整ってきた

- [Budget and usage management APIs now generally available](https://github.blog/changelog/2026-06-04-budget-and-usage-management-apis-now-generally-available)
- [API access to billing usage reports now generally available](https://github.blog/changelog/2026-06-04-api-access-to-billing-usage-reports-now-generally-available)

予算の作成・更新・削除、利用量の集計、コストセンター情報の取得、請求レポートの CSV 生成まで API 化が進みました。  
Enterprise 管理者や billing 担当にとっては、**UI でしかできなかった運用をスクリプト化できる** のが大きいです。

## まとめ

- **Developer Blogs** は、Learn MCP Server と Copilot の使い分けが .NET 開発に効く
- **GitHub Changelog** は、Copilot の実行系と billing/usage 系 API の強化が目立つ
- 今日は新機能の華やかさより、**AI を安全に・運用しやすく使うための土台が進んだ日** でした
