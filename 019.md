# 2026年6月2日分：Aspire 13.4 が前進、GitHub Copilot は運用前提が変わった

## 先に結論

2026年6月2日 JST 時点で、.NET 開発者にとって今日いちばん見ておきたいのは **Aspire 13.4** です。

これは .NET Blog の新着ではありませんが、アプリ開発・ローカル実行・デプロイ・運用の体験に関わる更新がまとまっていて、**普段 .NET を触る人ほど影響を受けやすい内容**でした。

一方で、GitHub Changelog は **既存ユーザーへの実務影響が大きい更新**が中心です。特に Copilot は、6月1日から全プランが GitHub AI Credits ベースの従量課金になった点をまず押さえておくのがよさそうです。

## Developer Blogs（.NET に興味がある人向け）

### Aspire 13.4 は .NET 読者にも重要

- [Aspire 13.4 is here](https://devblogs.microsoft.com/aspire/whats-new-aspire-13-4?WT.mc_id=DT-MVP-5004827)

今回の更新で大きいのは、Aspire が単なる新機能追加ではなく、**アプリホスト、CLI、運用補助、デプロイ先連携まで少しずつ実践向きに揃ってきた**ことです。

特に気になったポイントは次のとおりです。

- **TypeScript apphost が GA**
- **typed resource command arguments**
- **サーバー側 CLI のログ / テレメトリ検索**
- **Kubernetes / AKS デプロイの成熟**
- **Go / Bun の統合サポート**
- **ダッシュボードの AI Agents エントリポイント**
- **Foundry hosted agent サポートの改善**
- **CLI / doctor の改善**

.NET 読者にとって意味があるのは、Aspire が「.NET 専用の話」ではなく、**.NET を中心にしながら周辺技術も一緒に扱うための土台**になってきている点です。

たとえば、TypeScript apphost の GA や Go / Bun 統合は、バックエンドが .NET、フロントや補助サービスは別技術、という今どきの構成と相性がよさそうです。  
また、Kubernetes や AKS まわりの成熟は、ローカル開発だけでなく、その先のデプロイ先まで視野に入れやすくなる変化として見ておきたいです。

ログ / テレメトリ検索や CLI / doctor 改善も地味ですが重要です。日々の開発では、派手な新機能よりも **詰まったときに原因へ早くたどり着けるか** が効くからです。

それに加えて、**日本の開発現場でも無視しにくかった不具合が 13.4 で手当てされた**点も見逃せません。  
[Backchannel path is too long](https://github.com/microsoft/aspire/issues/15752) では、Aspire 13.2.0 以降、**Windows のユーザー名やホームディレクトリに非 ASCII 文字が入ると**、バックチャネル用の Unix Domain Socket パスが UTF-8 バイト長の上限を超えて失敗する問題が報告されていました。

この問題は日本語専用ではありませんが、日本語の氏名や全角文字を含む Windows ユーザー名では特に踏みやすい条件です。実際に issue でも、日本の enterprise ユーザーにとってアップグレードの妨げになっている、という指摘が出ていました。

修正は [Shorten backchannel socket paths](https://github.com/microsoft/aspire/pull/17226) として 13.4 向けにマージされていて、`~/.aspire/cli/backchannels/` から **より短い `~/.aspire/cli/bch/` 配下へソケット配置を詰める**ことで、非 ASCII を含むパスでも余裕を持たせる形です。  
つまり 13.4 は新機能だけでなく、**日本語環境を含む多言語 Windows 環境での実用性を一段上げたリリース**としても見ておく価値があります。

## GitHub Changelog（既存ユーザーへのインパクト重視）

### 既存ユーザーは Copilot の課金変更を最優先で確認

- [Updates to GitHub Copilot billing and plans](https://github.blog/changelog/2026-06-01-updates-to-github-copilot-billing-and-plans)

今回いちばん影響が大きいのはここです。

6月1日以降、**すべての Copilot プランが GitHub AI Credits ベースの従量課金**になりました。既存ユーザー目線では、これまでよりも「契約しているから使い放題」とは捉えにくくなり、**使い方と予算管理をセットで考える必要がある**更新です。

加えて、**Copilot code review は AI Credits に加えて GitHub Actions minutes も消費**します。ここは見落とすと、後で請求や利用枠の認識のずれにつながりやすそうです。

実務的に見ると、影響が大きいのは次のあたりです。

- 個人利用でも、使い方によってコスト感が変わる
- 組織利用では、コードレビュー運用が Actions minutes にも影響する
- **org / enterprise のユーザーレベル予算管理が GA** になり、統制しやすくなった
- 既存の Student / Pro / Pro+ 契約者は **Copilot Max へアップグレード可能**
- 一方で、**Student / Pro / Pro+ / Max の新規申込は引き続き停止中**

新機能の盛り上がりというより、**今使っている人ほど運用ルールを見直したくなる更新**だと受け止めるのがよさそうです。

### auto の評価モデル混在も確認しておきたい

- [Evaluation models in auto for individual plans](https://github.blog/changelog/2026-06-01-evaluation-models-in-auto-for-individual-plans)

個人向けの非エンタープライズユーザーでは、Copilot の **auto でのモデル選択に evaluation models が含まれる場合がある**とのことです。

既存ユーザーへの影響としては、いつもどおり auto を使っていても、**モデルの挙動や体感が変わる可能性がある**点がポイントです。  
もし安定性や期待する挙動を優先したいなら、GitHub Copilot の設定画面から無効化できる、と理解しておくと安心です。

- 設定場所: `/settings/copilot/features`

ここは「すごい新機能が来た」という話ではなく、**既存の auto 利用ユーザーが違和感を持ったときの確認ポイント**として覚えておくのがよさそうです。

## まとめ

今日の見どころは、次の 2 つでした。

- **.NET 開発者向けの実質的な注目記事は Aspire 13.4**
- **GitHub Copilot は既存ユーザーほど課金と設定変更の影響確認が必要**

特に GitHub 側は、機能追加というより **利用条件の前提が変わる更新**です。  
普段から Copilot を使っているなら、AI Credits、Actions minutes、予算設定、auto の評価モデル混在の 4 点は早めに見直しておくとよさそうです。
