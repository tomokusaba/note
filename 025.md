# 2026年6月8日分：.NET 目線で読む Microsoft Developer Blogs と GitHub Changelog

## 先に結論

今日は、**.NET 開発の土台を広げる Microsoft Developer Blogs** と、**既存ユーザーの運用に効く GitHub Changelog** が目立ちました。  
Developer Blogs では **Aspire の多 repo 展開** と **Foundry Local の強化** が実務寄りで、GitHub 側では **Copilot モデルの入れ替え** と **Copilot SDK GA** が特に重要です。

## Developer Blogs（.NET 目線で読む）

### Aspire は「広げる段階」に入っている

- [Aspire Multi-repo Rollout at Scale with Agentic AI](https://devblogs.microsoft.com/aspire/aspire-windows-365-part2/)

この記事は、Windows 365 が Aspire を **50+ リポジトリ** に広げていった話です。  
単に AI で自動化したのではなく、先に **SDK 固定、ヘルスチェック、再現性のあるテスト** をそろえてから、agentic AI で横展開しています。

`.NET` の現場で見ると、これはかなりわかりやすい教訓です。  
AI を入れる前に、**壊れにくい標準手順** を作っておくほうが、結果的に速いという話でした。

### Foundry Local は .NET から使いやすい方向に進んでいる

- [Accelerate Edge AI Development with Foundry Local](https://devblogs.microsoft.com/foundry/accelerate-edge-ai-development-with-foundry-local/)

Foundry Local 1.2.0 では、**Broader .NET compatibility**、**Linux ARM64 対応**、**WinML 2.0**、**WebGPU**、**キャンセル制御** が強化されています。  
さらに、GitHub Copilot CLI の音声入力が Foundry Local ベースで動いている点も面白く、**ローカル AI を実アプリに組み込む流れ** がかなり具体的になりました。

要するに、Edge AI が「試すもの」から **配れるもの** に寄ってきています。

## GitHub Changelog（既存ユーザーへの影響が大きいもの）

### GPT-5.2 系の廃止は見逃しにくい

- [GPT-5.2 and GPT-5.2-Codex deprecated](https://github.blog/changelog/2026-06-05-gpt-5-2-and-gpt-5-2-codex-deprecated/)

Copilot の多くの体験で **GPT-5.2 / GPT-5.2-Codex が廃止** されました。  
代替は **GPT-5.5** と **GPT-5.3-Codex** です。

モデル名を固定している運用や、社内手順書に古いモデル名が残っている環境では、**設定と説明文の見直しが必要** です。

### Copilot SDK が GA になった

- [Copilot SDK is now generally available](https://github.blog/changelog/2026-06-02-copilot-sdk-is-now-generally-available/)

Copilot SDK は、.NET を含む 6 言語で一般提供になりました。  
agent の計画、tool 実行、ファイル編集、ストリーミング、多段対話を自前で組まなくてよくなるので、**社内ツールに Copilot を埋め込む** ならかなり大きい更新です。

### Visual Studio の Copilot は「計画してから直す」方向へ進んだ

- [GitHub Copilot in Visual Studio — May update](https://github.blog/changelog/2026-06-04-github-copilot-in-visual-studio-may-update/)

Plan agent、Skills パネル、複数ファイル変更の要約差分など、Visual Studio の Copilot は **補完よりも作業単位の支援** に寄っています。  
既存ユーザーに効くのは、**大きめの変更を安全に進める導線** が増えたことです。

### Enterprise Teams は管理運用に効く

- [Enterprise Teams is now generally available](https://github.blog/changelog/2026-06-04-enterprise-teams-is-now-generally-available/)

Enterprise Teams の GA は、企業内で同じチームを org ごとに作り直す手間を減らします。  
権限、レビュー、ruleset bypass、SCIM、audit log まで含めて、**大規模運用を軽くする更新** です。

## まとめ

- **Developer Blogs** は、Aspire の多 repo 展開と Foundry Local の強化が .NET 目線で有用
- **GitHub Changelog** は、モデル廃止・Copilot SDK GA・Visual Studio 更新が特に重要
- 今日は、**AI を実務で使うための土台が一段進んだ日** でした
