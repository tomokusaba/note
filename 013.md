# 2026年5月27日分：.NET 開発者向け注目更新と GitHub Changelog まとめ

## 先に結論

5月27日まわりの更新は、**「AI にどう使われるかを前提に開発体験を整える話」**と、**GitHub 運用を自動化・統制しやすくする話**が中心でした。

特に気になったのは次の4点です。

- AI エージェント時代は、SDK や CLI の「人間向けに何となく使いやすい」だけでは足りない
- Visual Studio の Copilot 体験が、編集前の計画・レビュー重視に寄ってきた
- GitHub Copilot Memory の制御が増え、チーム運用で扱いやすくなった
- Code Quality や secret scanning の運用を、UI だけでなく API でも回しやすくなった

## Developer Blogs（.NET に興味がある人向け）

この日付帯の Developer Blogs では、**純粋な .NET 記事は少なめ**でした。  
そのため今回は、.NET そのものの新機能というより、**.NET 開発者にも影響が大きい話題**を拾っています。

### AI coding agents に技術をどう使わせるか

[How AI coding agents actually use your technology](https://devblogs.microsoft.com/blog/how-ai-coding-agents-actually-use-your-technology?WT.mc_id=DT-MVP-5004827) は、AI コーディングエージェントが実際にどう SDK / API / CLI / ツールを使うかを、かなり実務寄りに整理した記事でした。

重要なのは、失敗の多くが**ツール呼び出し以前の見えにくい段階**で起きるという点です。  
文脈の組み立て、ツール選択、引数解釈、出力の読み取りまで含めて考えないと、「呼ばれたかどうか」だけ見ても十分ではありません。

.NET 開発者にとっては、社内 CLI、MCP サーバー、API、SDK を作るときに、

- 簡潔で誤解しにくいツール説明
- 正しいスキーマ
- 機械が読み取りやすい出力
- 行動につながるエラーメッセージ

を整える重要性が、かなりはっきり見える内容でした。

### Visual Studio の Copilot が計画・レビュー寄りに進化

[Visual Studio May Update – Plan, Review, Refine](https://devblogs.microsoft.com/visualstudio/visual-studio-may-update-plan-review-refine?WT.mc_id=DT-MVP-5004827) では、編集前の探索と実装計画を行う Plan agent、skills 管理、コンテキスト使用量の可視化、会話要約、複数ファイル差分の要約などが紹介されています。

普段 Visual Studio で .NET 開発をしている人にとっては、  
**「まず書かせる」より「先に把握して、あとで直す」**方向へ Copilot の使い方が変わってきた、と見ると分かりやすそうです。

## GitHub Changelog（既存ユーザーへのインパクト重視）

### Copilot Memory の制御が増えた

[Copilot Memory has more controls for deletion, scope, and the Copilot CLI](https://github.blog/changelog/2026-05-26-copilot-memory-has-more-controls-for-deletion-scope-and-the-copilot-cli) は、既存ユーザーへの影響が大きい更新です。  
削除まわりの案内、リポジトリ単位のオフ設定、ユーザー設定とリポジトリ事実の区別、Copilot CLI の `/memory on|off|show` が追加されました。

「便利だけど、どこまで覚えるのかが気になる」という不安に対して、運用しやすい方向に進んだ印象です。

### GitHub Code Quality を API で有効化できる

[GitHub Code Quality: Repository Enablement API](https://github.blog/changelog/2026-05-26-github-code-quality-repository-enablement-api) は public preview で、github.com 向け、GHES は対象外です。  
PATCH / GET のセットアップ API により、既定セットアップの有効化・無効化、対象言語、runner 種別、設定取得を自動化できます。

対応言語に `csharp` が入っているので、C# を含む組織では**横展開を UI 手作業で回さなくてよくなる**のが大きいです。

### secret scanning の承認リクエストを整理しやすくなった

[Filter secret scanning approval requests by sort order and bypass status](https://github.blog/changelog/2026-05-26-filter-secret-scanning-approval-requests-by-sort-order-and-bypass-status) では、bypass / dismissal リクエストの並び替えと、`is_bypassed` クエリでの絞り込みが追加されました。

小さな改善に見えますが、secret scanning をすでに運用しているチームほど効く更新です。  
件数が増えたときの見通しが、かなり変わりそうです。

## まとめ

今回の更新は、派手な .NET 新機能の日ではありませんでした。  
ただし、**AI 時代の開発体験をどう設計するか**、**GitHub 運用をどう制御・自動化するか**という意味では、実務への影響が大きい日だったと思います。

.NET 開発者目線では、次の2つを意識しておくと良さそうです。

- 自分たちのツールや API を「AI が読んで使う前提」で見直す
- Visual Studio / GitHub の新しい計画・統制機能を、チーム運用にどう取り込むか考える
