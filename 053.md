## microsoft/aspire

.NET Aspire は分散アプリケーション開発を支援するフレームワーク。今回は VS Code 拡張機能のリリースと、CLI のテスト容易性を高めるリファクタリングがマージされました。

### リリース

1. **[#18528] Prepare VS Code extension release v1.16.0**
   VS Code 拡張機能のバージョンを v1.15.0 → v1.16.0 へバンプするリリース準備がマージされました。CLI インストールのシェル非依存化、AppHosts ツリービューでの単一グループノードのフラット化、Aspire VS Code 拡張ファンネルの改善、ワークスペースビュー切り替え時のグローバル AppHost クリア修正が含まれます。
   [PR を見る](https://github.com/microsoft/aspire/pull/18528)

### リファクタリング

3. **[#18483] Replace direct OS checks with IEnvironment in CLI**
   CLI 内の OS 検出を `IEnvironment` 抽象化経由に統一するリファクタリングがマージされました。`IEnvironment` に `IsWindows()` / `IsLinux()` / `IsMacOS()` メソッドを追加し、`[SupportedOSPlatformGuard]` 属性付きで実装しています。本番実装の `HostEnvironment` は `OperatingSystem.IsXxx()` に委譲し、テスト向けには `TestEnvironment.CreateWindows()` / `CreateLinux()` / `CreateMacOS()` ファクトリメソッドを提供。約30ファイルが更新され、`CertificateServiceTests` のOS スキップ条件7件が削除されて全プラットフォームで実行可能になりました。
   [PR を見る](https://github.com/microsoft/aspire/pull/18483)

## dotnet/runtime

.NET ランタイム本体。JIT バグ修正・WASM インフラ更新・パフォーマンス改善・cDAC 診断強化など低レイヤーの変更が多数マージされました。

### JIT バグ修正

1. **[#129917] JIT/wasm: keep EH frame slots at the bottom after frame alignment**
   #129815 のフレームアライメントパディングが WASM EH スロットの固定オフセットをずらし、EH ディスパッチを破壊していた問題を修正。EH フレームスロットをフレームアライメント後も末尾に維持するよう変更されました。
   [PR を見る](https://github.com/dotnet/runtime/pull/129917)

3. **[#129914] JIT: fix stale VN in redundant branch opts dominating-branch simplification**
   冗長ブランチ最適化の dominating-branch 簡略化処理で、古い VN（Value Number）が使われてしまうバグが修正されました。
   [PR を見る](https://github.com/dotnet/runtime/pull/129914)

5. **[#129903] JIT/wasm: pass type-context arg for LDVIRTFTN calls**
   WASM 上で `LDVIRTFTN` 命令を呼び出す際に型コンテキスト引数が渡されていなかった問題が修正されました。
   [PR を見る](https://github.com/dotnet/runtime/pull/129903)

### WebAssembly / プラットフォーム

7. **[#129925] [11.0p6] Upgrade Emscripten to 5.0.6 & LLVM to 23**
   .NET 11 の Preview 6 ブランチで Emscripten が 5.0.6、LLVM が 23 にアップグレードされました。WASM ビルドツールチェーンの最新化です。
   [PR を見る](https://github.com/dotnet/runtime/pull/129925)

9. **[#129906] Disable OpenBSD execute-only for NativeAOT stubs**
   OpenBSD における NativeAOT スタブへの execute-only フラグが無効化されました。OpenBSD プラットフォームでの NativeAOT 動作安定化に寄与します。
   [PR を見る](https://github.com/dotnet/runtime/pull/129906)

### パフォーマンス改善

11. **[#129911] AES-KWP Performance Improvements for Apple**
    Apple プラットフォームにおける AES-KWP（AES Key Wrap with Padding）の性能が改善されました。Apple の CryptoKit ネイティブ API を活用した最適化とみられます。
    [PR を見る](https://github.com/dotnet/runtime/pull/129911)

13. **[#128365] Add AvxVnni.V512 hardware intrinsics**
    AVX-VNNI-INT8 命令セットの 512 ビット幅バリアントに対応するハードウェア組み込み関数が追加されました。ML ワークロードや整数演算の高速化が期待できます。
    [PR を見る](https://github.com/dotnet/runtime/pull/128365)

19. **[#129884] Avoid stripping IL of non-async Task-returning methods**

## microsoft/agent-framework

.NET / Python Agent Framework は AI エージェント構築のためのフレームワーク。今回は Python の 1.10.0 メジャーリリースと、.NET 側の承認フロー修正がマージされました。

### リリース

1. **[#6753] Python: bump package versions for 1.10.0 release**
   Python 向けの 1.10.0 リリースがマージされました。`agent-framework`・`agent-framework-core`・`agent-framework-openai`・`agent-framework-foundry` がいずれも 1.9.0/1.8.2 → 1.10.0 へバンプされています。**破壊的変更**として、FileSkillsSource のリファクタリング、ファイルアクセスツールへの承認要件追加、HarnessAgent ループ統合、FileMemoryProvider の移植が含まれます。実験的 API を利用しているコードは更新が必要です。
   [PR を見る](https://github.com/microsoft/agent-framework/pull/6753)

### バグ修正

3. **[#6674] .NET: Enforce ApprovalRequiredAIFunction in GitHub Copilot provider via OnPreToolUse hook**
   .NET の GitHub Copilot プロバイダーで `ApprovalRequiredAIFunction` が正しく機能しない問題が修正されました。Copilot SDK のネイティブ事前実行フック `SessionConfig.Hooks.OnPreToolUse` を使って承認フローを実装し、承認必須ツールには `"ask"` を返して `OnPermissionRequest` にルーティングします。呼び出し元が独自フックを設定している場合はそちらを優先し、警告ログを出力します。
   [PR を見る](https://github.com/microsoft/agent-framework/pull/6674)
