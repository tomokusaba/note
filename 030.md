self-hosted runner を使っているなら、**「あとで更新する」では回らなくなる日付が見えた**ので、先に運用を確認しておくのが大事です。

## GitHub Changelog（既存ユーザーへの影響が大きいもの）

### self-hosted runner の最小バージョン enforcement が再開する

- [GitHub Actions: Minimum version enforcement timeline for self-hosted runners](https://github.blog/changelog/2026-06-12-github-actions-minimum-version-enforcement-timeline-for-self-hosted-runners/)

GitHub は、self-hosted runner のバージョン要件 enforcement を再開します。

実務上のポイントは 3 つです。

1. **runner 2.329.0 以上**でないと、configure や再登録ができない  
2. 新しい runner リリースごとに、**30日以内**に更新しないと job を実行し続けられない  
3. **auto-update 有効**で update service に到達できる runner なら、この条件を基本的に満たしやすい

全面適用の時期は次のとおりです。

- **2026年7月31日**：GitHub Enterprise Cloud with Data Residency
- **2026年9月25日**：GitHub Enterprise Cloud

対象は **github.com / Enterprise Cloud** で、**現時点では GHES は対象外** です。

self-hosted runner を固定運用しているチームや、通信制限のある環境では特に注意が必要です。  
「いま動いているか」ではなく、**30日以内に更新し続けられるか** を先に見ておきたい更新でした。

## まとめ

- 今回の確認範囲では、Microsoft Developer Blogs 側で追加で拾うものはなかった
