# OSS Daily: 2026-06-26 注目PRまとめ

2026年6月25日（UTC）にマージされた OSS プロジェクトのプルリクエストを、リポジトリごとにまとめました。

今回は **microsoft/fluentui-blazor**・**microsoft/aspire**・**dotnet/runtime**・**microsoft/agent-framework** の4リポジトリで活発な動きがありました。バグ修正から大型リファクタリング、パフォーマンス改善、AI エージェント基盤の機能追加まで、幅広いアップデートが一日でマージされています。ぜひ気になるリポジトリのセクションから読んでみてください。

---

## 1. microsoft/fluentui-blazor

**本日のハイライト:** Toast サービスの大規模リファクタリングが入り、任意の Razor コンポーネントを Toast 内に埋め込めるようになりました。バグ修正も2件マージされています。

- [#4964 \[dev-v5\]\[Layout\] Assign IconTitle only when it's not set](https://github.com/microsoft/fluentui-blazor/pull/4964)
  `FluentLayoutHamburger` の `IconTitle` パラメーターが壊れていたバグを修正しました。null 合体代入演算子（`??=`）を使い、未設定時のみ値を代入するように変更しています（Issue #4963 の対応）。

- [#4962 \[dev-v5\]\[Autocomplete\] Disable autocomplete suggestions](https://github.com/microsoft/fluentui-blazor/pull/4962)
  `FluentAutocomplete` でブラウザのオートフィル候補がポップアップメニューと重なる問題を修正しました。`autocomplete="false"` では効果がなかったため `autocomplete="off"` に変更しています。

- [#4953 \[dev-v5\] Update project files for .NET 8 support](https://github.com/microsoft/fluentui-blazor/pull/4953)
  プロジェクトファイルを .NET 8 対応に更新しました。Release ビルドで `net8.0` / `net9.0` / `net10.0` をターゲットとするよう調整されています。

- [#4950 \[dev-v5\] Toast Service Refactoring](https://github.com/microsoft/fluentui-blazor/pull/4950)
  Toast サービスの大規模リファクタリングです。`INotificationService` を導入して MessageBar と Toast を共通化し、任意の Razor コンポーネントを Toast 内に埋め込めるようになりました。`ShowSuccessToastAsync` などの便利メソッドが追加されたほか、`ResultTiming` オプションで await タイミングを細かく制御できるようになっています。

## 5. microsoft/aspire

**本日のハイライト:** AppHost 探索の大幅高速化（cold 実行 24秒→12秒）と、ASP.NET プロジェクトへの HTTPS 証明書自動適用という実用的な改善が同日にマージされました。

- [#18499 Add field IDs to issue templates for prefill support](https://github.com/microsoft/aspire/pull/18499)
  Issue フォームのテキストエリアフィールドに安定した `id` 属性を追加しました。GitHub のプリフィル URL でフィールドを事前入力できるようになり、製品内フィードバック機能（#18377）への対応が進んでいます。

- [#18491 \[Automated\] Update Microsoft Foundry Models](https://github.com/microsoft/aspire/pull/18491)
  Microsoft Foundry モデルディスクリプター（`FoundryModel.Generated.cs`）の自動更新です。

- [#18487 Fix spurious error logging for expected backchannel disconnects in `aspire run`](https://github.com/microsoft/aspire/pull/18487)
  `aspire run` で AppHost 終了時のバックチャネル切断が誤ってエラーレベルのログとして出力されていた問題を修正しました。切断を正常なストリーム完了として扱うように変更し、テレメトリのノイズも削減されています。

- [#18481 Configure HTTPS certificates for project resources](https://github.com/microsoft/aspire/pull/18481)
  ASP.NET プロジェクトリソースが Aspire の HTTPS 証明書設定を直接使用できるようになりました。各プロジェクトで個別に Kestrel マッピングを書かなくても、TLS エンドポイントに証明書が自動適用されます。

- [#18436 Speed up AppHost discovery by skipping MSBuild evaluation for non-AppHost projects](https://github.com/microsoft/aspire/pull/18436)
  `aspire ls` などの AppHost 探索を大幅に高速化しました。全 `.csproj` に対して MSBuild 評価を行っていた問題を解消し、XML を直接パースする軽量フィルターで AppHost らしくないプロジェクトを事前に除外する仕組みを導入しています。**cold 実行が 24秒→12秒、warm 実行が 6秒→2秒**に改善されました。

## 7. dotnet/runtime

**本日のハイライト:** WebAssembly への Vector128 高速パス導入と JSON エスケープ処理への SearchValues 活用など、パフォーマンス改善が複数マージされた一日でした。

- [#129839 Fix blob type in CopyWithPrivateKey for ML-DSA](https://github.com/dotnet/runtime/pull/129839)
  Windows Insider ビルドで一部の ML-DSA テストが失敗する問題を修正しました。`CopyWithPrivateKey` でシード Blob をインポートする際のブロブタイプ指定が誤っていたため修正されています。

- [#129838 \[wasm\] Enable Vector128 fast paths on Wasm via PackedSimd](https://github.com/dotnet/runtime/pull/129838)
  WebAssembly PackedSimd を使った `Vector128` 高速パスを有効化しました。hex/Guid、UTF-8、SearchValues、Teddy、Adler32/XXH3 などへの適用により、Wasm 上でのスループットが向上します。

- [#129837 Use assembly simple name in ReadyToRun log when path is empty](https://github.com/dotnet/runtime/pull/129837)
  パスが空の場合に ReadyToRun ログでアセンブリ簡易名を使用するよう修正しました。

- [#129831 Pass correct search path to DllImportResolver for default flags on Mono](https://github.com/dotnet/runtime/pull/129831)
  Mono 上でデフォルトフラグを使用する際に `DllImportResolver` へ正しい検索パスが渡されていなかった問題を修正しました。

- [#129824 Update Alpine infra images to 3.24](https://github.com/dotnet/runtime/pull/129824)
  インフラ用 Alpine イメージを 3.24 に更新しました。

- [#129818 Root DotNetRuntimeContractDescriptor export against linker GC in NativeAOT](https://github.com/dotnet/runtime/pull/129818)
  NativeAOT でリンカー GC によって `DotNetRuntimeContractDescriptor` のエクスポートが消去されないよう、明示的に保持するように変更しました。

- [#129816 JIT/wasm: bail R2R for calls passing 16-byte SIMD by value](https://github.com/dotnet/runtime/pull/129816)
  Wasm の JIT で 16 バイト SIMD を値渡しするコールについて ReadyToRun を回避するように修正しました。

- [#129815 \[wasm R2R\] Align 8-byte stack locals and keep frames 16-aligned](https://github.com/dotnet/runtime/pull/129815)
  wasm R2R で 8 バイトのスタックローカルをアライメントし、フレームを 16 バイトアラインに保つ修正です。

- [#129814 Fix windows-arm64 timeout floor to use numeric comparison](https://github.com/dotnet/runtime/pull/129814)
  windows-arm64 のタイムアウト下限を数値比較で判定するように修正しました。

- [#129811 \[release/11.0-preview6\] Add Stream wrappers for memory and text-based types](https://github.com/dotnet/runtime/pull/129811)
  メモリおよびテキストベースの型向けに Stream ラッパーを追加しました（`release/11.0-preview6` へのバックポート）。

- [#129809 Use span.Min/Max helpers to speed up SearchValues.Create](https://github.com/dotnet/runtime/pull/129809)
  `SearchValues.Create` のパフォーマンス改善のため、span の `Min`/`Max` ヘルパーメソッドを活用するように変更しました。

- [#129789 \[S390X\] fix utf8->utf16 conversion test case failures](https://github.com/dotnet/runtime/pull/129789)
  S390X アーキテクチャで UTF-8→UTF-16 変換のテストが失敗する問題を修正しました。

- [#129781 Use SearchValues to scan for characters to escape with default Json options](https://github.com/dotnet/runtime/pull/129781)
  デフォルト JSON オプションでエスケープ対象文字をスキャンする際に `SearchValues` を活用してパフォーマンスを向上させました。

- [#129779 Use `Process.RunAndCaptureText` in src/tests child-process helpers](https://github.com/dotnet/runtime/pull/129779)
  テストの子プロセスヘルパーで `Process.RunAndCaptureText` を使用するよう統一しました。

- [#129768 Add ZstandardDecompressionOptions for symmetric decoder configuration](https://github.com/dotnet/runtime/pull/129768)
  Zstandard デコーダーの設定を対称的に行うための `ZstandardDecompressionOptions` クラスを追加しました。

- [#129767 Add "load instructions" step to code-review skill](https://github.com/dotnet/runtime/pull/129767)
  コードレビュースキルに「命令のロード」ステップを追加しました（開発インフラ整備）。

- [#129747 Check for ComInterop case for async variants in crossgen2](https://github.com/dotnet/runtime/pull/129747)
  crossgen2 で非同期バリアントの ComInterop ケースを検出するよう修正しました。
