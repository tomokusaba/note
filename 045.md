## microsoft/aspire — 安定性・DX・CI の三拍子

この日もっとも活発だったリポジトリ。20件のPRから特に目を引いたものを取り上げます。

1. **PR #18431 — Fix user secrets escaping `&` and `+`**
   `&` → `\u0026`、`+` → `\u002B` にエスケープされてパスワードが壊れるという、地味に痛いバグが修正されました。接続文字列やAPIキーを user secrets に保存している場合は要注目です。

3. **PR #18383 — Cap rendered items in UrlsColumnDisplay**
   URLが数百件あるリソースで `FluentOverflow` の全子DOM要素をレンダリングしてしまい、UIスレッドがブロック → Blazor Server の SignalR 接続が切れるという問題を修正。大規模環境の Aspire Dashboard で刺さる問題でした。

5. **PR #18393 — Detect miscased CLI options**
   `aspire run --AppHost` のような大文字小文字違いのオプションを検出して正しいスペルを提案するようになりました。`TreatUnmatchedTokensAsErrors = false` のコマンドでは従来サイレントに無視されていたのを、親切なエラーに昇格。CLIのDXが一段階上がりました。

7. **PR #18410 — First-party Nix package for Aspire CLI**
   NixOS ユーザー待望の公式 Nix パッケージ。バージョン付き GitHub リリースアーカイブを SHA512 SRI ハッシュでピン留めする設計で、再現性を担保しています。

9. **PR #18260 — VS Code: Open terminal action**
   Aspire VS Code 拡張のリソースツリーのコンテキストメニューに「ターミナルを開く」アクションを追加。`aspire terminal attach` をGUIから直接呼び出せるようになり、デバッグ体験が向上します。

11. **PR #18269 — Improve Azure provisioning recovery**
    長時間実行中の状態表示改善、AppHost 再起動後のリカバリー強化、ARMデプロイ状態や Key Vault のソフトデリート対応など、実運用で詰まりやすい箇所がまとめて手当てされました。

13. **PR #18421 — OIDC federated auth for VM Sizes workflow**
    Azure VM サイズ更新の自動化ワークフローに OIDC フェデレーション認証を採用。長命なシークレットを排除する CI セキュリティのベストプラクティスが自リポジトリにも適用されています。

## dotnet/runtime — ランタイム深部の品質強化

こちらも20件。低レベルな修正が多く、ランタイムチームの底力が光ります。

1. **PR #129728 / #129744 — Fix ValueType.GetHashCode for nested generic fields**
   ネストされたジェネリックフィールドを持つ値型の `GetHashCode` が誤った値を返すバグを修正。main と release/10.0 の両方にバックポートされています。ハッシュ系のバグはコレクション操作で静かに壊れるため、影響範囲は侮れません。

3. **PR #129714 — Fix GC hole when return is hijacked**
   GCサスペンションのためにメソッドリターンがハイジャックされた際に生じる GC ホールを修正。NativeAOT のバックポート (#129711) も同日マージ。GC の安全性に関わるクリティカルな修正です。

5. **PR #129722 — JIT: invert oversize loops with own bounds checks**
   過大なループを独自境界チェック付きで反転するJIT最適化。ループの形状を変えることで範囲外チェックの除去を促進するパターンです。

7. **PR #129475 — Port System.Net.Http to OpenBSD**
    OpenBSD 向けの `System.Net.Http` 移植。.NET のマルチプラットフォーム対応が着実に広がっています。

9. **PR #129563 — Ensure SIMD loads/stores are marked as used**
    SIMD組み込み関数使用時にSIMD関連のロード・ストア・間接参照が適切に「使用済み」としてマークされない問題を修正。最適化フェーズで誤って削除されるリスクを排除します。

## microsoft/agent-framework — エージェント基盤の設計とセキュリティ

.NET と Python の両スタックで活発な更新が続いています。

1. **PR #6679 — ADR 0029: Skills over MCP design options**
    MCP リソーステンプレートスキルと直接スキル参照という2つのアーキテクチャオプションを検討した ADR が追加。skills-over-MCP スペックのリリース待ちで決定は保留中ですが、設計の方向性が文書化されました。

3. **PR #6672 — Emit empty sections for available_resources/scripts**
    スキルにリソースやスクリプトがない場合でも明示的に空セクションを出力するように変更。モデルが存在しないセクションをハルシネーションするケースへの対策です。地道ですが効いてくる修正。

5. **PR #6624 — Per-run refreshable MCP auth headers sample**
    リクエストごとにヘッダーをリフレッシュできる MCP 認証パターンのサンプルを追加。IAM/OBO トークン（有効期限 ~1時間）やテナントごとのヘッダー切り替えに対応できる設計を実証しています。

7. **PR #6599 — [BREAKING] Require approval for file-access tools (Python)**
    ファイルアクセスツールの自動承認を廃止し、書き込み・削除操作にホストの承認を必須化。読み取りのみ自動承認継続。エージェントが勝手にファイルを変更するリスクを抑えるセキュリティ強化で、BREAKING CHANGE ながら方向性は正しいと感じます。

9. **PR #6635 — Change A2A default session store to NoopAgentSessionStore**
    A2A ホスティングのデフォルトセッションストアを Noop（永続化なし）に変更し、永続化はオプトイン方式へ。デフォルトを安全・シンプル側に倒す設計判断です。

## まとめ

この日のPR群を眺めると、いくつかのテーマが浮かびます。

**バグ修正の奥深さ**：`ValueType.GetHashCode` の汎用フィールド問題や GC ホール修正など、.NET ランタイムの根幹に関わる修正が複数マージされました。表面上は地味でも、放置すれば不可解な挙動として現れる類のバグです。

**DXへの継続投資**：Aspire の CLI オプション誤りの提案、VS Code ターミナルアクション、Nix パッケージ対応など、「使いやすくする」改善が途切れません。

**セキュリティの内製化**：agent-framework のファイルアクセス承認必須化、CI ワークフローへの OIDC 採用など、セキュリティをデフォルトの選択肢として組み込む動きが加速しています。

日々のOSSコミットの積み重ねが、エコシステム全体の信頼性を底上げしていることを改めて実感する一日分でした。
