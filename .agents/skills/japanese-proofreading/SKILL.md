# 日本語推敲スキル

このスキルは、日本語で書かれた文章（特に note / ブログ記事）を読みやすく整えるためのチェックリストと修正フォーマットを提供します。文章のスタイルそのものは [`write-note-article`](../write-note-article/SKILL.md) に従い、その上で**機械的に検出可能な問題**を本スキルで潰します。

---

## 1. 推敲チェックリスト

以下の観点を上から順に全文へ適用してください。

### 1.1 文体の統一

- です・ます調 / である調を混在させない。
- 一人称は本文の既存方針に揃える。特に指定がなければ「私」を基本にする。
- 体言止めは強調したい箇所だけに使う。

### 1.2 表記ゆれ

代表的な揺れは、プロジェクト内の既存記事または一般的な技術文書表記に寄せる。

| 推奨 | 避ける | 備考 |
|------|--------|------|
| ユーザー | ユーザ | JIS Z 8301 / Microsoft 日本語スタイルに準拠 |
| サーバー | サーバ | 同上 |
| コンピューター | コンピュータ | 同上 |
| ブラウザー | ブラウザ | 同上 |
| Web | web / WEB | 固有名詞・略語は表記を固定 |
| GitHub | Github / github | 固有名詞は公式表記 |
| .NET | dotnet / Dotnet | 文中での名称は `.NET`、識別子は `dotnet` |
| 例えば | たとえば | 文書内で統一されていればどちらでも可 |
| および | 及び | ひらく |
| 〜できます | 〜する事ができます | 冗長表現を避ける |

### 1.3 冗長表現の削減

| 修正前 | 修正後 |
|--------|--------|
| 〜することができます | 〜できます |
| 〜という事ができます | 〜と言えます |
| 〜を行います | 〜します |
| 〜の場合においては | 〜の場合は |
| 〜において | 〜で |
| 〜に関して | 〜について |
| まず最初に | まず / 最初に |
| 後で後悔する | 後悔する |

### 1.4 一文の長さ

- 目安は 60〜80 文字 / 1 文。100 文字を超えたら分割を検討する。
- 「が、」で文が続いている場合は、逆接でないなら句点で切る。

### 1.5 受動態の濫用

- 「〜される」「〜られる」が 2 文連続、または 1 段落に 3 つ以上あれば能動態への書き換えを検討する。
- 例: 「設定が読み込まれます」→「設定を読み込みます」

### 1.6 句読点・記号

- 句読点は「、」と「。」を基本にする。
- 1 文中の読点は 2〜3 個までを目安にする。
- 半角英数の前後には半角スペース 1 つを入れる（例: `Azure を使う`）。
- 括弧は全角（）を基本に、コード・識別子内は半角 `()` を使う。
- 三点リーダは「…」を使う。「...」は避ける。
- 感嘆符・疑問符の後ろは全角スペース 1 つ（行末は不要）。

### 1.7 カタカナ語と英字の使い分け

- 広く定着しているカタカナ語: ユーザー、サーバー、デプロイ、ログ、コンテナー
- 製品名・固有名詞は原綴: Azure、Kubernetes、PostgreSQL、Visual Studio Code
- 略語は初出時に展開する（例: CI（継続的インテグレーション））。

### 1.8 専門用語の初出時の補足

- 初めて出る専門用語には和訳または英語表記を併記する。
- 例: 「アクセシビリティ（利用しやすさ）」。

### 1.9 見出し・箇条書き

- 見出しは原則として体言止め、または「〜する」「〜とは」形式にする。
- 箇条書きは粒度を揃える。
- 番号付きリストは手順・順序に使う。並列概念は `-` を使う。

### 1.10 「ら抜き」「い抜き」「さ入れ」

- 見れる / 食べれる / 出れる → 見られる / 食べられる / 出られる
- 〜してます → 〜しています
- 〜させていただきます（過剰敬語） → 〜します / 〜いたします

## 2. 出力フォーマット

推敲結果は「修正前 / 修正後 / 理由」3 列の Markdown 表で返します。理由欄では本スキル §1 の項番を参照してください。

```markdown
## 推敲結果

### サマリ
- チェック項目: 文体 / 表記ゆれ / 冗長 / 受動態 / 句読点 / その他
- 検出件数: N 件（うち必須修正 N 件、推奨修正 N 件）

### 修正提案

| # | 修正前 | 修正後 | 理由 |
|---|--------|--------|------|
| 1 | この機能は内部で処理が行われます。 | この機能は内部で処理を行います。 | §1.5 受動態の濫用 |
```

長い文章を扱う場合は、セクション（`##`）ごとに上記表を分けて出力します。

---

## 3. やってはいけないこと

- 元文の意味を変える
- 著者の文体を消す
- 専門用語を勝手に平易な言葉に置換する
- コードブロック・引用ブロック内の文章を書き換える（コメントを除く）
- note 固有の Markdown 形式を変更する
