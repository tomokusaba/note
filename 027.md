# 2026年6月10日分：.NET 11 Preview 5 と GitHub Copilot の重要更新

## 先に結論

6月10日は、**.NET 11 Preview 5 の公開** と、**GitHub Copilot 周辺の運用変更** が同時に進んだ日でした。  
Microsoft Developer Blogs では .NET の中核機能が着実に前進し、GitHub Changelog では既存ユーザーに直撃する更新がいくつか並びました。

今日は、**.NET に興味がある人向けの新着** と、**GitHub の既存ユーザーに影響が大きい更新** に絞ってまとめます。

## Developer Blogs（.NET 目線で読む）

### .NET 11 Preview 5 は、普段使いの改善が多い

- [.NET 11 Preview 5 is now available!](https://devblogs.microsoft.com/dotnet/dotnet-11-preview-5/)

.NET 11 Preview 5 は、派手な単発機能というより、**日々の開発体験を底上げする改善** が多い印象です。

目立つ点を挙げると、こんな感じです。

- `System.Text.Json` が JSON Lines を扱えるようになった
- LINQ に full outer join が入った
- X25519 の鍵交換が追加された
- async の停止処理や GC の最適化が進んだ
- file-based app から別の C# ファイルを参照できるようになった
- ビルド時に SDK の脆弱性 / EOL をチェックできる
- `dotnet new` に MCP Server テンプレートが追加された
- コンソールアプリで `System.Net.Http.Json` が既定で入る

アプリ側でも、Blazor SSR のクライアント側検証、QuickGrid の非対話実行、EF Core の `dotnet ef` 対応など、実務で触る部分の改善が入っています。

要するに、.NET 11 は「新機能を増やす」よりも、**SDK・ランタイム・Web・EF Core を横断して、開発の摩擦を減らす** 方向に進んでいます。

## GitHub Changelog（既存ユーザーへの影響が大きいもの）

### Claude Fable 5 が Copilot に追加された

- [Claude Fable 5 is generally available for GitHub Copilot](https://github.blog/changelog/2026-06-09-claude-fable-5-is-generally-available-for-github-copilot/)

Claude Fable 5 は、Copilot の新しい上位モデルとして使えるようになりました。  
Pro+ / Max / Business / Enterprise で使え、VS Code、Visual Studio、Copilot CLI、cloud agent、github.com、モバイル、JetBrains、Xcode、Eclipse まで広く展開されています。

ただし、**データ保持が必要** で、Business / Enterprise では管理者が明示的に有効化する必要があります。  
モデル追加そのものより、**運用ポリシーと課金条件の確認** が重要です。

### 第三者 coding agent のセキュリティ検証が GA

- [Security validation for third-party coding agents](https://github.blog/changelog/2026-06-09-security-validation-for-third-party-coding-agents/)

Claude や OpenAI Codex など、GitHub Copilot 以外の coding agent が作ったコードにも、Copilot cloud agent と同じセキュリティ検証が入るようになりました。

検証内容は、CodeQL、依存関係の Advisory Database 照合、secret scanning です。  
問題が見つかれば、PR を確定する前に agent 側で修正を試みます。

これは、**AI が書いたコードの品質チェックが agent の種類を問わず標準化されてきた** という意味で、かなり重要です。

### 休眠リポジトリにも定期コードスキャン

- [Periodic code scanning of inactive repositories](https://github.blog/changelog/2026-06-09-periodic-code-scanning-of-inactive-repositories/)

6か月以上 push や PR のないリポジトリでも、code scanning の定期実行を 30 日ごとに回せるようになりました。対象は code scanning default setup を使っている組織です。

放置されがちなリポジトリでも脆弱性の見逃しを減らせるので、**古い repo を多く抱えている組織ほど効く更新** です。

## まとめ

- **.NET 11 Preview 5** は、開発体験の摩擦を減らす改善が中心
- **GitHub Copilot** は、モデル追加とセキュリティ運用の両方が動いた
- 既存ユーザーにとっては、**機能追加よりも設定・運用の見直しが大事** な日でした
