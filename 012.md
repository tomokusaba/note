## Developer Blogs（.NET に興味がある人向け）

### 1. Copilot 活用記事：まず「モード」ではなく「タスク」で選ぶ

[Doing More with GitHub Copilot as a .NET Developer](https://devblogs.microsoft.com/dotnet/doing-more-with-github-copilot/?WT.mc_id=DT-MVP-5004827) は、.NET 開発で Copilot を使うときの実践ガイドです。

要点は、Visual Studio / VS Code / CLI / cloud agent を「どれが上位か」ではなく、**今の作業に合う面で使い分ける**ことです。  
チャットで理解・設計を進めるケースと、エージェントで実装・修正まで進めるケースを具体プロンプト付きで整理していて、日々の開発にそのまま移植しやすい内容でした。

### 2. .NET MAUI：Android 向け Material 3 を実運用で試せる段階へ

[Give Your .NET MAUI Android Apps a Material 3 Makeover](https://devblogs.microsoft.com/dotnet/dotnet-maui-material-3/?WT.mc_id=DT-MVP-5004827) は、.NET MAUI 10 の Android UI 更新です。

`.csproj` に `<UseMaterial3>true</UseMaterial3>` を追加するだけで有効化でき、主要コントロールの見た目が Material 3 に寄ります。  
記事時点では `Microsoft.Maui.Controls` 10.0.60 以降推奨で、Android 専用機能である点や、まだ専用対応が進行中の領域も明示されていました。

## GitHub Changelog（既存ユーザーへのインパクト重視）

### 1. PR 上でカバレッジ確認（Public Preview）

[Code coverage on pull requests is now in public preview](https://github.blog/changelog/2026-05-26-code-coverage-in-pull-requests-is-now-in-public-preview) により、PR 画面で集約カバレッジ率を直接確認できるようになりました。

既存ユーザーへの影響が大きい理由は、**レビュー導線そのものにテスト網羅のシグナルが統合される**ためです。  
導入には Cobertura レポートのアップロードと、GitHub Apps / Actions での `code-quality:write` 権限が必要です（Team / Enterprise Cloud、プレビュー期間は無料）。

### 2. Copilot モデルの組織別ルール（Public Preview）

[Target Copilot models to organizations with model rules](https://github.blog/changelog/2026-05-26-target-copilot-models-to-organizations-with-model-rules) で、Enterprise 単位の一括設定だけでなく、**組織ごとに許可モデルを出し分け**できるようになりました。

既存運用で「部門ごとに使わせるモデルを変えたい」ニーズがある場合、影響はかなり大きいです。  
Copilot Business / Enterprise 向けの更新で、既定モデル管理 UI の改善も同時に入っています。

### 3. GitHub Classroom：新規サインアップ停止

[GitHub Classroom sign-ups are no longer available](https://github.blog/changelog/2026-05-26-github-classroom-sign-ups-are-no-longer-available) は、教育用途の既存ユーザーに直接効く変更です。

新規サインアップは停止済みで、既存利用者は継続利用可能。  
ただし 2026-08-28 に完全移行予定とされており、**利用継続中の教育チームは移行計画を前倒しで持つ必要がある**更新です。
