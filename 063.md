## dotnet/runtime

.NET ランタイムでは、LINQ の性能回復や WebAssembly / ARM64 周りの安定化、テスト再有効化が中心でした。

### パフォーマンス改善

1. **[#130138] LINQ の MinInteger / MaxInteger パスを復元**
   LINQ で整数列に対する `MinInteger` / `MaxInteger` の最適化パスが回帰で失われていたため、性能に敏感なケースでの挙動が復旧しました。
   [PR を見る](https://github.com/dotnet/runtime/pull/130138)

### テスト / 品質改善

1. **[#130136] ML-DSA PKCS8 エクスポートテストを再有効化**
   以前無効化されていた PKCS8 export のテストを再度有効化し、回帰検知の網を戻しました。
   [PR を見る](https://github.com/dotnet/runtime/pull/130136)

3. **[#130120] Arm64 hwintrinsic テストを再有効化**
   ARM64 の hwintrinsic テストが無効化されていたため、関連機能の回帰確認ができるようになりました。
   [PR を見る](https://github.com/dotnet/runtime/pull/130120)

### WebAssembly / プラットフォーム

1. **[#130131] codegenwasm のビルドブレークを修正**
   WebAssembly 用コード生成のビルドブレークを修正し、関連ビルドの安定性が回復しました。
   [PR を見る](https://github.com/dotnet/runtime/pull/130131)

3. **[#130111] browser/CoreCLR で -O3 を有効化**
   Emscripten の更新後に最適化レベルを上げられなかった browser/CoreCLR 構成で、より高い最適化を有効化しました。
   [PR を見る](https://github.com/dotnet/runtime/pull/130111)

5. **[#130101] SIMD16 store-indirect / parameters の R2R を回避**
   WebAssembly の SIMD16 ケースで不正な動作を避けるため、R2R の適用を制限しました。
   [PR を見る](https://github.com/dotnet/runtime/pull/130101)

7. **[#130090] cDAC の HFA / HVA 対応を拡張**
   Windows ARM64 と Linux ARM/ARM64 で diagnostics の ARGITER と zero-known-issues を有効化し、cDAC / 診断支援の対象範囲が広がりました。
   [PR を見る](https://github.com/dotnet/runtime/pull/130090)

### ドキュメント / API

1. **[#130117] `unsafe` 修飾子の追加は破壊的変更であることを明記** 「破壊的変更」
   既存コードに `unsafe` 修飾子を追加することが破壊的変更となることを、ドキュメント上で明示しました。互換性を確認する際の判断材料として有用です。
   [PR を見る](https://github.com/dotnet/runtime/pull/130117)

   承認ハーネス機能の既定動作を構成可能にし、カスタマイズしやすい shell tool も追加されました。運用環境ごとに承認ポリシーを合わせやすくなっています。

9. **[#6869] .NET サンプルを改善**
