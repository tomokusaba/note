### MCP を CI パイプラインで使う：.NET ビルド診断の自動化

**[MCP Beyond the Chat Window: Build Diagnostics in CI](https://devblogs.microsoft.com/dotnet/mcp-build-diagnostics-workflows/)**（Jan Krivanek, Yuliia Kovalova）

MSBuild のバイナリログ（binlog）を AI エージェントが読み解く **Microsoft Binlog MCP Server** の続報です。今回はチャット画面での対話ではなく、**GitHub Actions CI パイプライン上での無人実行**にフォーカスしています。

- PR ビルドが失敗すると、エージェントが自動的に binlog を解析し、根本原因を PR コメントとして投稿する（ファイル名・行番号・修正案つき）
- `microsoft/testfx` リポジトリで実際に稼働中のワークフローを公開ソースで確認できる
- MCP ツールは合計 **38 種類**に拡張済み（前回紹介の 15 種から大幅増）。ターゲット実行理由の調査（`binlog_target_reasons`）、2 ビルド間の差分比較（`binlog_compare`）、自動診断（`binlog_diagnose`）など実務向けのツールが充実
- 自動解析はあくまで **advisory（助言）** であり、マージゲートには関与しない設計

binlog を人手でダウンロードしてログを読む作業をエージェントに委ねられる実践的な内容です。

## GitHub Changelog（既存ユーザーにインパクトが強い重要更新）

### Claude Sonnet 5 が GitHub Copilot で GA

**[Claude Sonnet 5 is generally available for GitHub Copilot](https://github.blog/changelog/2026-06-30-claude-sonnet-5-is-generally-available-for-github-copilot)**

Anthropic 最新の Sonnet クラスモデル **Claude Sonnet 5** が Copilot で正式リリースされました。

- 対象プラン：Copilot Pro / Pro+ / Max / Business / Enterprise
- 利用可能サーフェス：VS Code、Visual Studio、JetBrains、Xcode、Eclipse、Copilot CLI、GitHub Copilot App、github.com、GitHub Mobile
- CLI スタイルのタスクで特に高いパフォーマンスを発揮し、低 effort レベルでの低レイテンシが特徴
- **Zero Data Retention (ZDR)** 対応。Enterprise / Business 管理者はモデルポリシーから有効化
- 課金は Usage Based Billing（provider list pricing）

### コードカバレッジによる PR マージ保護が追加

**[GitHub code coverage merge protection for pull requests](https://github.blog/changelog/2026-06-30-github-code-coverage-merge-protection-for-pull-requests)**

ブランチルールセットに **テストカバレッジ閾値によるマージ保護**を設定できるようになりました（パブリックプレビュー）。

- カバレッジが指定した最低パーセンテージを下回る、またはデフォルトブランチから一定以上低下した場合にマージをブロック
- まず **evaluate モード** で影響を確認し、準備できたら **active モード** に切り替え可能
- GitHub Enterprise Cloud および Team の GitHub Code Quality ユーザーが対象。プレビュー期間中は無料

### Copilot Agent が JetBrains AI Assistant にネイティブ統合

**[Copilot Agent is now available in JetBrains AI Assistant](https://github.blog/changelog/2026-06-30-copilot-agent-is-now-available-in-jetbrains-ai-assistant)**

JetBrains AI Assistant の **エージェントピッカーから GitHub Copilot を直接選択**できるようになりました。

- Copilot モデルの切り替えや reasoning depth の調整が AI チャット内で完結
- 複数ステップのコーディング作業をエージェントに委任可能
- 近日中に NES（Next Edit Suggestions）やスキル呼び出しへの対応も予定

### Dependabot が .npmrc を自動推測しなくなる（要対応）

**[Dependabot no longer infers .npmrc](https://github.blog/changelog/2026-06-30-dependabot-no-longer-infers-npmrc)**

Dependabot が lockfile の `resolved` URL から `.npmrc` 設定を自動推測する動作が**廃止**されました。プライベート npm レジストリを使用しているリポジトリは対応が必要です。

- `dependabot.yml` のレジストリ定義に `scope` プロパティを追加することで、Dependabot が正しい `.npmrc` を自動生成するようになります
- リポジトリにコミット済みの `.npmrc` が存在し、`scope` を設定していない場合は従来どおり動作します
- github.com の全ユーザーが対象。GHES 3.23 にもバックポート予定

### クローズ済みセキュリティアラートのデータ保持ポリシーが導入予定

**[Upcoming cloud data retention policy for closed security alerts](https://github.blog/changelog/2026-06-30-cloud-data-retention-policy-for-closed-security-alerts)**

**2026 年 8 月 25 日**より、クローズ済み Dependabot セキュリティアラートにデータ保持ポリシーが適用されます。

- クローズから **2 年以上経過したアラートはアーカイブストレージ**に移行し、UI・API での通常アクセスが不可になります
- アーカイブされたアラートは Enterprise / Organization / リポジトリ管理者・セキュリティマネージャーが **CSV でダウンロード**可能
- オープンなアラート、およびクローズから 2 年未満のアラートは影響なし
- GHES には適用されません（github.com / GHEC のみ）

8 月 25 日までに REST API で古いアラートを取得・保存している場合は影響範囲を確認してください。
