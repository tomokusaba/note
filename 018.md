# 2026年6月1日分：.NET 本体は静か、Foundry の月次更新は実務寄り

## 先に結論

今日の新着は、**.NET Blog 本体には直近 1 日で大きな更新は見当たらず**、Developer Blogs では **Microsoft Foundry の月次更新**がいちばん実務に近い話題でした。

.NET の読者目線で見ると、C# や SDK から触れられる範囲が広がったこと、ローカル実行や評価基盤、運用まわりが前に進んだことがポイントです。  
一方、GitHub Changelog では、**直近 1 日で既存ユーザーにすぐ影響する新着は見当たりませんでした**。静かな日ですが、それ自体が確認結果として価値があります。

## Developer Blogs

### .NET Blog 直系の新着はなし

執筆時点で、[.NET Blog](https://devblogs.microsoft.com/dotnet/?WT.mc_id=DT-MVP-5004827) の直近の表示では、過去 1 日の新着は見当たりませんでした。  
今日は「.NET 本体の大きな発表を追う日」というより、周辺の AI 開発基盤を押さえる日です。

### Foundry 月次更新は .NET 開発者にも関係が深い

今回の [What's new in Microsoft Foundry | May 2026](https://devblogs.microsoft.com/foundry/whats-new-in-microsoft-foundry-may-2026?WT.mc_id=DT-MVP-5004827) で、.NET 開発者に関係が深いのは、次の点です。

- **azure-ai-projects 2.2.0 が .NET を含む複数言語に展開**
- **external agents / skills / toolboxes / routines / optimization jobs** など、SDK から扱える範囲が広がった
- **trace-based evaluation** により、外部エージェントやホスト型エージェントの評価がしやすくなった
- **Managed VNET が GA** になり、運用面の安心感が増した
- **project-level cost attribution** で、利用コストの見え方が改善
- **Foundry Local 1.1 / 1.2** では、live audio transcription、text embeddings、Qwen 3.5 Vision、Linux ARM64、ONNX Runtime 1.26 などが追加

ここで大事なのは、「新モデルが増えた」ことよりも、**.NET から AI 機能を組み込むときの足場が少しずつ揃ってきた**ことです。  
特に、評価・ローカル実行・ネットワーク分離・コスト把握が同時に進んでいるので、試作だけでなく、運用を意識した検証もしやすくなっています。
