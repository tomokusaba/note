# 2026年6月7日分：Developer Blogs と GitHub Changelog の1日分まとめ

## 先に結論

執筆時点では、[.NET Blog](https://devblogs.microsoft.com/dotnet/) に直近1日の新着はありませんでした。  
その代わり、[Developer Blogs](https://devblogs.microsoft.com/) では **Aspire の大規模展開** と **Foundry Local の更新** が .NET 目線で読みやすく、[GitHub Changelog](https://github.blog/changelog/) では **Copilot のモデル更新・SDK GA・Visual Studio 更新** が既存ユーザーに効く内容でした。

## Developer Blogs（.NET 目線で読む）

### 複数リポジトリへ広げるなら、先に「再現性」を固める

- [Aspire Multi-repo Rollout at Scale with Agentic AI](https://devblogs.microsoft.com/aspire/aspire-windows-365-part2)

この投稿は、Windows 365 の Aspire 導入を **50以上のリポジトリへ広げた話** です。  
ポイントは、AI で一気に広げたのではなく、先に **テスト・ヘルスチェック・SDK/ツールの固定** で再現性を作ってから、agentic AI でロールアウトを自動化していることです。

ざっくり言うと、流れはこうです。

1. まずサービスを壊れにくくする  
2. 次に、チェックと修正の手順を固定する  
3. その後に、AI で横展開する

.NET 開発でも、既存サービスの改善や複数 repo への共通パターン展開にそのまま当てはまる内容でした。

### Foundry Local は .NET からも使いやすい方向に進んでいる

- [Accelerate Edge AI Development with Foundry Local](https://devblogs.microsoft.com/foundry/accelerate-edge-ai-development-with-foundry-local)

Foundry Local の 1.2.0 は、**ローカル AI を本番寄りに使うための土台** がかなり増えています。  
特に .NET 目線では、**Broader .NET compatibility**、Windows ML 2.0、WebGPU、キャンセル制御、Linux ARM64 対応が目立ちます。

注目点は次のあたりです。

- **40+ 言語の音声認識**
- **Linux ARM64 対応**
- **ダウンロードと推論のキャンセル**
- **WinML 2.0**
- **WebGPU execution provider**

Copilot CLI の音声入力が Foundry Local で動いている、という話もあり、ローカル推論をアプリに組み込む実例としても読みやすい記事でした。

## GitHub Changelog（既存ユーザーへの影響が強いもの）

### GPT-5.2 系モデルは廃止、置き換えが必要

- [GPT-5.2 and GPT-5.2-Codex deprecated](https://github.blog/changelog/2026-06-05-gpt-5-2-and-gpt-5-2-codex-deprecated)

Copilot の多くの体験から **GPT-5.2 / GPT-5.2-Codex が廃止** されました。  
代替はそれぞれ **GPT-5.5** と **GPT-5.3-Codex** です。

Copilot Chat、inline edits、agent mode、code completions を使っているなら、**モデル指定や運用手順を見直す必要があります**。

### Copilot SDK が GA になり、.NET でも正式に使いやすくなった

- [Copilot SDK is now generally available](https://github.blog/changelog/2026-06-02-copilot-sdk-is-now-generally-available)

Copilot SDK が一般提供になり、.NET でも `GitHub.Copilot.SDK` として使えます。  
agent の計画、ツール実行、ファイル編集、ストリーミング、多段対話までを **自前で組まなくてよくなる** のが大きいです。

既存ユーザー視点では、単なる新機能よりも **「Copilot を自分のアプリや社内ツールに組み込める」** ことのインパクトが大きい更新でした。

### Visual Studio の Copilot は「計画してから直す」方向へ進んだ

- [GitHub Copilot in Visual Studio — May update](https://github.blog/changelog/2026-06-04-github-copilot-in-visual-studio-may-update)

Visual Studio の Copilot では、**Plan agent**、**Skills パネル**、**複数ファイル変更の要約差分** が追加されました。  
既存ユーザーに効くのは、補完よりも **「大きめの変更を安全に進める導線」** が整ってきた点です。

### Enterprise Teams の GA は管理者向けにかなり大きい

- [Enterprise Teams is now generally available](https://github.blog/changelog/2026-06-04-enterprise-teams-is-now-generally-available)

Enterprise Teams が GA になり、enterprise で一度定義したチームを複数 org にまたがって使えるようになりました。  
SRE やセキュリティチームの運用、ruleset bypass、SCIM、audit log まで含めて、**Enterprise 管理の手間を減らす更新** です。

## まとめ

- **Developer Blogs** は、Aspire の大規模展開と Foundry Local の更新が .NET 目線で有用
- **GitHub Changelog** は、モデル廃止・Copilot SDK GA・Visual Studio 更新が特に重要
- 今日は「新機能の派手さ」より、**既存の開発フローを壊さずに AI を組み込む土台が進んだ日** でした
