## Microsoft Developer Blogs（.NET 目線で読む）

### Aspire VS Code 拡張 13.4：TypeScript AppHost が GA、CodeLens で手軽に操作できるように

- [Aspire 13.4 is here!](https://devblogs.microsoft.com/aspire/aspire-13-4-is-here/?WT.mc_id=DT-MVP-5004827)

Aspire の VS Code 拡張がバージョン 13.4 になりました。日常の操作性とマルチ言語対応の両面で大きく前進したリリースです。

ポイントは3つです。

1. **VS Code エディタ上で AppHost の CodeLens アクションが使えるようになった**。Run / Stop / Restart / Open Dashboard / View Logs を、Aspire パネルを開かずにエディタから直接実行できます
2. **TypeScript AppHost が GA（一般提供）になった**。C# を使わずに TypeScript だけで AppHost を記述できるようになり、フロントエンド寄りのチームでも導入しやすくなっています
3. **Python・Java・Go の AppHost もプレビューとして追加**。マルチ言語プロジェクトへの Aspire 適用の幅が一気に広がりました

公開時刻を UTC→JST 変換すると **2026年6月16日 JST 内に入る新着** です。

## GitHub Changelog（既存ユーザーへの影響が大きいもの）

### GitHub Code Quality が 2026年7月20日に GA（有料化）

- [GitHub Code Quality generally available July 20, 2026](https://github.blog/changelog/2026-06-16-github-code-quality-generally-available-july-20-2026/)

**7月20日から有料化**されます。既存ユーザーへの影響が最も大きい変更です。

ポイントは3つです。

1. **月額 $10 per active committer**（有効なコミッター1人あたり）の課金が始まる
2. GA 後の主な機能として、findings / scoring / ルールセット統合 / セキュリティ品質概要 / 組織全体デプロイ / 品質ゲート（maintainability・reliability・coverage）の PR 適用が提供される
3. **課金を避けたい場合は、7月20日より前にリポジトリで Code Quality を無効化する必要がある**

意図せず課金される可能性があるため、利用中のリポジトリを今すぐ確認しておくことをおすすめします。

公開時刻は **2026-06-16 UTC** なので、**日本時間では 2026年6月16日**の新着です。

---

### 組織単位で GitHub Code Quality を一括有効化できるようになった

- [Organization-level enablement for GitHub Code Quality](https://github.blog/changelog/2026-06-16-organization-level-enablement-for-github-code-quality/)

上記の GA と合わせて、**組織設定から全リポジトリへ一括で Code Quality を適用・解除できる機能が public preview として追加されました。**

ポイントは3つです。

1. **組織設定の Security タブに「Code quality」セクションが追加**された
2. **組織全リポジトリに対して一括で有効化・無効化できる**。個別リポジトリを一つひとつ操作する手間がなくなります
3. 対象は **GitHub Enterprise Cloud・GitHub Team プラン**。GHES（GitHub Enterprise Server）は対象外

GA 後の課金を踏まえると、この一括無効化機能は特に「まだ使う予定のない組織」にとって重要な手段になりそうです。

公開時刻は **2026-06-16 UTC** なので、**日本時間では 2026年6月16日**の新着です。

---

### GitHub Models が新規顧客に対して利用不可になった

- [GitHub Models is no longer available to new customers](https://github.blog/changelog/2026-06-16-github-models-is-no-longer-available-to-new-customers/)

GitHub Models の退役（retirement）プロセスが始まりました。

ポイントは3つです。

1. **新規顧客（これまで利用実績のない組織・Enterprise）は GitHub Models を使えなくなった**
2. **既存ユーザーは引き続き利用可能**。playground・API・モデルアクセスは現時点では維持される
3. **代替として Azure AI Foundry が推奨されている**。移行を検討している場合は早めに確認しておくとよいでしょう

今後の完全廃止スケジュールは別途通知される予定です。既存ユーザーも今のうちに移行計画を立てておくのが安心です。

公開時刻は **2026-06-16 UTC** なので、**日本時間では 2026年6月16日**の新着です。

## まとめ

- 今回より **JST 基準** の直近24時間で確認する方式に変更しました
- **Aspire VS Code 拡張 13.4** では TypeScript AppHost が GA になり、CodeLens で手軽に AppHost を操作できるようになりました
- **GitHub Code Quality は 2026年7月20日に有料化**。課金を避けるには 7月20日 前に無効化が必要です
- **組織単位で Code Quality を一括有効化・無効化できる機能**が public preview として追加されました
- **GitHub Models は新規顧客への提供を停止**。退役プロセスが始まりました。既存ユーザーへの影響は現時点では軽微ですが、移行先として Azure AI Foundry が推奨されています
