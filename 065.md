## dotnet/runtime

.NET ランタイムでは、JIT の最適化改善と、Wasm・AOT・テスト周りの整理が中心でした。

### JIT / コンパイラ改善

1. **[#130186] JIT: initialize emitCurIG at declaration**
   JIT の emitCurIG 初期化を宣言時に行うようにして、コンパイラ内部状態の扱いをより明確にしています。低レベルの安定性改善として位置づけられます。
   [PR を見る](https://github.com/dotnet/runtime/pull/130186)

3. **[#130181] JIT: Enable tail await optimization in async versions in tier 0**
   async 版の tier 0 で tail await 最適化を有効化し、非同期コードの実行効率を改善する変更です。小さな非同期メソッドでの性能向上が期待されます。
   [PR を見る](https://github.com/dotnet/runtime/pull/130181)

### WebAssembly / プラットフォーム

1. **[#130153] [Wasm/RyuJit] treat all try side-entries as try entries for throw helpers**
   WebAssembly の RyuJit で、throw helper に関わる try side-entry をすべて try entry として扱うようにして、例外経路の挙動を整えています。
   [PR を見る](https://github.com/dotnet/runtime/pull/130153)

3. **[#129762] Enable ICF on Linux**
   Linux でも ICF を有効化し、バイナリサイズの圧縮や最適化の適用範囲を広げています。配布サイズに関心のある環境での効果が期待されます。
   [PR を見る](https://github.com/dotnet/runtime/pull/129762)

### テスト / 静的解析

1. **[#130132] Enable trim/AOT analyzers on System.Text.Json source generator tests**
   System.Text.Json の source generator テストに trim / AOT analyzers を有効化し、生成コードの静的解析カバレッジを強化しています。長期的な品質改善に寄与します。
   [PR を見る](https://github.com/dotnet/runtime/pull/130132)

3. **[#100729] Add Decimal32, Decimal64, Decimal128**
   Decimal32 / Decimal64 / Decimal128 のサポート追加で、より幅広い精度要件に対応できるようになります。新しい数値表現の扱いに関心がある開発者にとって重要な追加です。
   [PR を見る](https://github.com/dotnet/runtime/pull/100729)

Agent Framework では、.NET と Python の双方で API の意味合いを見直す変更が続いています。

   ShellPolicy の allow / deny の解釈を Python 側と揃えるため、.NET での振る舞いを変更する破壊的変更です。既存のポリシー実装を持つユーザーは影響範囲を確認しておくとよさそうです。

5. **[#6892] .NET: Return 404 for a response created against a nonexistent conversation**

7. **[#6890] .NET: Validate Foundry toolbox name is a single path segment before building the proxy URL**
