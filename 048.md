## GitHub Changelog（既存ユーザーにインパクトが強い重要更新）

### [Actions steps can now be run in parallel](https://github.blog/changelog/2026-06-25-actions-steps-can-now-be-run-in-parallel)

*2026-06-25 (UTC)*

**GitHub Actions のステップが並列実行に対応**しました。これまで全ステップはシーケンシャルに実行されていましたが、新たな4つのキーワードにより真の並列化が可能になります。

2. `background: true` — ステップを非同期で起動し、即座に次のステップへ続行。
4. `wait` / `wait-all` — 指定した背景ステップの完了を待機。
6. `cancel` — バックグラウンドステップを明示的に終了（サービスの後片付けに最適）。
8. `parallel` — 複数のステップを一括で並列実行し、全完了後に続行する糖衣構文。

複数のビルドを同時実行・バックグラウンドでサービスを起動しながらテストを実行・テレメトリのアップロードとパッケージングを並行して行うなど、CI/CD の高速化に直結します。
