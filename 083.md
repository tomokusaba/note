   [PR を見る](https://github.com/dotnet/runtime/pull/130954)

5. **#131301 Add managed cDAC type wrapper scaffolding**
   `CLRDataTypeDefinition` と `CLRDataTypeInstance` の COM ラッパーの土台を追加するスキャフォールディングです。フォールバックが有効な場合はレガシー実装へすべて委譲し、無効な場合は `E_NOTIMPL` を返すのみで、この時点ではコントラクトに基づく実装は導入していません。
   [PR を見る](https://github.com/dotnet/runtime/pull/131301)

7. **#131273 Fix in-proc crash reporter frame assert.**
   インプロセスクラッシュレポーターのスタックウォーク処理が、`CrawlFrame` が `FrameLess` かどうかを確認せずに `GetRelOffset` を呼び出しており、デバッグビルドでアサートが発生していた不具合を修正しています。
   [PR を見る](https://github.com/dotnet/runtime/pull/131273)

9. **#131220 Add on-demand in-proc crash report generation.**
   ランタイムが稼働中の任意のタイミングでクラッシュレポートを生成できる、オンデマンド API を追加しています。致命的シグナル経路とは独立して呼び出せるようにすることで、ユーザー登録のネイティブエラーハンドラーからレポート生成機構だけを呼び出せるようにする狙いがあり、シグナル経路とオンデマンド経路は共有ライターへの同時アクセスを避けるため相互排他制御されます。
   [PR を見る](https://github.com/dotnet/runtime/pull/131220)

### バグ修正

1. **#130088 Fix Linux crash loading recursive Nullable static-abstract value types**
   `Nullable<T>` の `T` が static abstract インターフェースメンバーを持つ再帰的制約付き値型である場合に、Linux 上の CoreCLR がロード時にクラッシュしていた不具合を修正しています。System V 構造体分類が未解決のネストされた値型メタデータの存在を前提としてアサートしていた箇所を、保守的なフォールバックへ変更し、再現シナリオに基づく回帰テストも追加されました。
   [PR を見る](https://github.com/dotnet/runtime/pull/130088)

3. **#123234 Fix FTP/SSL data stream closure causing protocol violation in .NET 10**
   過去のリファクタリング（#106451）により SSL 使用時に `SslStream` が正しくクローズされなくなり、.NET 10 で FTPS のアップロード／ダウンロードがプロトコル違反エラーになっていた回帰を修正しています。SSL のグレースフルシャットダウンを先に行ってからタイムアウト付きでソケットをクローズする実装に直し、Docker ベースの Enterprise テストスイートも新規に追加されました。
   [PR を見る](https://github.com/dotnet/runtime/pull/123234)

5. **#131195 Fall back to NTLM in managed SPNEGO when Kerberos credentials are missing on Unix**
   マネージド SPNEGO 実装が Kerberos を楽観的機構として提示したものの、利用可能な KDC/TGT がなく `GSS_S_NO_CRED` が返る場合、OpenBSD（Heimdal）以外の Unix では NTLM へのフォールバックが行われず認証全体が失敗していた問題を修正しています。`UnknownCredentials` を全 Unix プラットフォームで回復可能な失敗として扱うようにし、Linux（MIT krb5）環境でのマネージド NTLM 認証失敗を解消しました。
   [PR を見る](https://github.com/dotnet/runtime/pull/131195)

7. **#130119 Fix config binder source generator emitting invalid tuple instantiation code**
   バインド対象にならないプロパティ経由でのみ到達する型に対して、コンフィグバインダーのソースジェネレーターが不要な `BindCore` ヘルパーを生成してしまっていた問題を修正しています。コンストラクター引数に対応するプロパティだけは除外対象から外すことで、コンパイルエラーを避けながら生成コードを削減しました。
   [PR を見る](https://github.com/dotnet/runtime/pull/130119)

9. **#130525 Fix EventSource WriteEvent varargs overloads dropped from EventPipe**
   `WriteEvent` の varargs 経路が型付きイベント経路とは別のプロバイダーレベルキーワードフィルタリングを行っており、一部のキーワード／セッションの組み合わせでイベントが欠落する不具合を修正しています。ETW と EventPipe それぞれの有効化判定を型付き経路と同じ方式に揃え、不要になったフィルタリング処理も削除しました。
   [PR を見る](https://github.com/dotnet/runtime/pull/130525)

11. **#131283 Update ad-hoc signing in HostModel for changed `codesign` behaviour (code directory page size) in macOS 26+**
    macOS 26 で `codesign` が arm64/arm64_32 の Mach-O オブジェクトを 16 KiB のコードディレクトリページサイズでハッシュ化するよう変更されました。これに合わせ、HostModel のマネージド ad-hoc 署名実装も Mach-O の CPU タイプからページサイズを判定するよう修正しています。
    [PR を見る](https://github.com/dotnet/runtime/pull/131283)

13. **#130118 Configuration Binder SG: ignore unresolvable metadata types to prevent ambiguous/missing-type emissions**
    直接参照されていない推移的アセンブリの型がエラーシンボルとして解決される場合に、コンフィグバインダーのソースジェネレーターがコンパイル不能なコードを生成してしまう問題を修正しています。エラー型を含むメンバー（ネストされた配列要素やジェネリック引数を含む）を再帰的に検出してバインド対象から除外し、スキップ時には利用者に伝わるよう `SYSLIB1101` 警告を出すようにしました。
    [PR を見る](https://github.com/dotnet/runtime/pull/130118)
