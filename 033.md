## 先に結論

今日は **日本時間の6月16日** ですが、掲載時刻は **PDT/PST 基準** で直近24時間を確認しました。  
その結果、**Microsoft Developer Blogs では Visual Studio Blog の新着が1件あり、GitHub Changelog でも Copilot 運用に効く新着が1件ありました。**

## Microsoft Developer Blogs（.NET 目線で読む）

### Visual Studio の見た目を細かく変えられるようになった

- [Make Visual Studio look the way you want](https://devblogs.microsoft.com/visualstudio/make-visual-studio-look-the-way-you-want/)

Visual Studio 2026 の **Theme colors** から、Fluent の色トークンを IDE 内で直接調整できるようになりました。  
拡張機能や JSON ファイルを毎回探さなくても、**テーマごとに個別の色を保存**できます。

ポイントは 3 つです。

1. **Theme colors** 画面から検索して色を変えられる
2. 変更は **テーマ単位** で保存される
3. JSON を共有すれば、同じ見た目をチームでも再現しやすい

公開時刻は **2026-06-15 14:00 UTC** なので、**日本時間では 6月15日 23:00 頃**。  
今回の 24 時間枠に入る新着です。

## GitHub Changelog（既存ユーザーへの影響が大きいもの）

### Copilot の usage metrics がより実態に近くなった

- [Copilot usage metrics now include more of your active users](https://github.blog/changelog/2026-06-15-copilot-usage-metrics-now-include-more-of-your-active-users/)

Copilot usage metrics API のレポートで、**クライアント側の信号だけでなくサーバー側のテレメトリも使う** ようになりました。  
その結果、これまで取りこぼしていた **active user** がレポートに反映されやすくなります。

既存ユーザーへの影響が大きいのは、次の点です。

1. **DAU の見え方が実態に近づく**
2. **billing や activity log とのズレが減る**
3. クライアント都合で落ちていたユーザーが **集計から消えにくい**

公開時刻は **2026-06-15 14:30 PDT** なので、**日本時間では 6月16日 6:30 頃**。  
今回の直近24時間にしっかり入っています。
