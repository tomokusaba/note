## GitHub Changelog

まず日々の PR さばきに直結する更新を見て、その後に管理者向けの予算管理アップデートを確認します。

### New pull requests dashboard is now generally available

- `github.com/pulls` の新しいダッシュボードが一般提供になりました。
- Inbox で、レビュー待ちの PR、修正が必要な PR、マージ準備ができた PR、merge queue に入っている PR をまとめて確認しやすくなっています。
- 保存済みビューのカスタマイズ、デフォルトフィルター、オートコンプリート、AND / OR / ネストした高度な検索、キーボード操作にも対応しています。
- 検索では `team-review-requested-user` と `review-involves` が追加され、`author` では agent-created PRs も扱えるようになりました。
- 普段 `is:open is:pr review-requested:@me` のような検索を使っているなら、まずは `github.com/pulls` を起点にレビュー導線を見直すと効果が出やすそうです。
- 参考: [New pull requests dashboard is now generally available](https://github.blog/changelog/2026-07-09-new-pull-requests-dashboard-is-now-generally-available/)

### Per-user states for multi-user budgets in the REST API

- GitHub Enterprise Cloud の enterprise owners と billing managers は、マルチユーザー予算に対する各ユーザーの進捗と上限を 1 つの REST API エンドポイントで取得できるようになりました。
- 利用率での絞り込み、特定ユーザーの確認、消費量順の並び替えに対応しています。
- enterprise 全体に適用する universal multi-user budget と、cost center にスコープされた multi-user budget の両方をサポートします。
- 予算の見え方を手作業で追っていた場合は、社内レポートやアラートの自動化に組み込みやすくなります。
- 参考: [Per-user states for multi-user budgets in the REST API](https://github.blog/changelog/2026-07-10-per-user-states-for-multi-user-budgets-in-the-rest-api/)

## まとめ

- PR レビュー運用では、新しい Pull Requests ダッシュボードを使う前提で検索条件と保存ビューを整理すると効率が上がりそうです。
- Enterprise Cloud の予算管理では、ユーザー単位の消費状況を REST API で取れるようになり、監視とレポートの自動化が進めやすくなりました。
- 実務では、まず Pull Requests ダッシュボードの保存ビューを見直し、Enterprise Cloud 管理者は per-user states API を既存レポートに組み込めるか確認するとよさそうです。
