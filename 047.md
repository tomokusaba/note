# 2026-06-25 OSS Changelog：Microsoft/dotnet/M5Stack エコシステム最新動向

2026年6月24日（JST 基準・過去1日分）にマージされた Pull Request をリポジトリ別に振り返ります。対象は **fluentui-blazor・fluentui・aspire・dotnet/runtime・agent-framework・StackChan** の6リポジトリ。CI/CD 安定化・クロスプラットフォーム対応・AOT 改善など多様なテーマで開発が動いた一日でした。

## microsoft/fluentui-blazor — パイプライン修正と Autocomplete 改善

v5 系リリースへ向けたメンテナンスが続きます。

## microsoft/aspire — CI 安定化・DX 改善・Orleans 拡張

James Newton-King 氏を中心に CI 基盤の堅牢化が集中して進みました。

1. **PR #18469 — Add 'Connection reset by peer' to transient CI failure retry patterns**
   ネットワーク由来のフレーキーな CI 失敗に対処。「Connection reset by peer」エラーをリトライ対象として登録しました。

3. **PR #18468 — Fix Playwright download transient errors failing the build**
   Playwright のバイナリダウンロード時に発生するトランジェントエラーでビルドが落ちる問題を修正。

5. **PR #18463 — Fix CA1310: Add StringComparison to StartsWith calls in Heartbeat.cs**
   コード分析警告 CA1310 に対応。`StartsWith` に `StringComparison` を明示することで、カルチャーに依存しない安全な文字列比較を実現。

7. **PR #18462 — [Automated] Update Microsoft Foundry Models**
   Foundry モデルの自動更新 PR。最新モデル定義をリポジトリに取り込みます。

9. **PR #18453 — Make Playwright browser provisioning resilient to download timeouts**
   Playwright ブラウザのプロビジョニング処理をダウンロードタイムアウトに対して耐性を持つよう改善。CI 環境の信頼性向上。

11. **PR #18451 — Fix flaky WaitForLegacyDetachedStartupStabilityAsync\_RetriesV2ProbeUntilChildExits test**
    子プロセス終了まで V2 プローブを再試行するシナリオで間欠的に失敗していたテストを修正。

13. **PR #18424 — Replace Environment.GetEnvironmentVariable in CLI with IEnvironment abstraction**
    CLI 内で直接呼び出していた `Environment.GetEnvironmentVariable` を `IEnvironment` インターフェース経由に置き換え。テスタビリティの大幅な向上が期待できます。

15. **PR #18256 — Enable keyboard navigation on dashboard scroll containers**
    ダッシュボードのスクロールコンテナでキーボードナビゲーションを有効化。アクセシビリティ基準への対応。

17. **PR #13630 — feat: add custom Orleans provider type annotation**
    Orleans カスタムプロバイダーの型アノテーションを追加。Aspire の Orleans サポートを拡充します。

## dotnet/runtime — JIT・Mono AOT・OpenBSD・パフォーマンス

特に Mono AOT 周りで Pavel Savara 氏、WebAssembly 周りで Andy Ayers 氏の活躍が目立ちました。

1. **PR #129788 — Fix IsKnownNonNull assert for SIMD base addresses**
   SIMD ベースアドレスに対する `IsKnownNonNull` アサートの誤判定を修正。JIT の正確性改善。

3. **PR #129783 — Ship stripped cdac AOT binary and its symbols in the transport package**
   ストリップ済み cdac AOT バイナリとシンボルファイルをトランスポートパッケージに同梱。診断・デバッグ体験の向上が期待されます。

5. **PR #129777 — Remove dead `fIs80Plus` parameter from `IMDInternalImport::GetUserString`**
   内部 API から長年使われていなかった `fIs80Plus` パラメータを削除。デッドコードのクリーンアップ（Copilot SWE Agent によるマージ）。

7. **PR #129770 — Implement KeySizeInBytes on AesGcm and AesCcm**
   `AesGcm` と `AesCcm` に `KeySizeInBytes` プロパティを実装。暗号化 API の使いやすさが向上（Copilot SWE Agent によるマージ）。

9. **PR #129765 — Obsolete AsnEncodedData.RawData.set**
   `AsnEncodedData.RawData` のセッターを廃止予定（Obsolete）としてマーク。不変性を促進する API 整理。

11. **PR #129763 — Use volatile access for CoreCLR dictionary layout publication**
    CoreCLR ディクショナリレイアウトの公開に volatile アクセスを使用。スレッド安全性の強化（Copilot SWE Agent によるマージ）。

13. **PR #129749 — Fix alignment of 8-byte thread statics on direct TLS path**
    直接 TLS パスで 8 バイトのスレッドスタティックが正しくアライメントされない問題を修正。EgorBo 氏による JIT/ランタイムバグフィックス。

15. **PR #129748 — Narrow Mono AOT typeload error cleanup**
    Mono AOT における型ロードエラーのクリーンアップ処理を改善。エラーハンドリングの精度向上。

17. **PR #129742 — Fix comparison-with-wider-type CodeQL issue**
    CodeQL が検出した「より広い型との比較」問題を修正。セキュリティ品質の維持。

19. **PR #129729 — Fix building unboxing and real generic method targets at the same time**
    アンボクシングターゲットと実ジェネリックメソッドターゲットを同時にビルドする際のクラッシュを修正。NativeAOT の安定性向上。

21. **PR #129716 — JIT/wasm: more EH related fixes**
    WebAssembly 向け JIT の例外ハンドリング関連の追加修正。WASM EH サポートの品質向上が続きます。

23. **PR #129715 — [mono] Defer NEWOBJ type-load failures to runtime in AOT mode**
    AOT モードで `NEWOBJ` 命令による型ロード失敗をランタイム時まで遅延させる変更。AOT の柔軟性向上。

25. **PR #129713 — [mono] Keep in-flight exception alive across LLVM resume unwind**
    LLVM の resume unwind 処理中に飛行中の例外オブジェクトが GC に回収されないよう修正。

27. **PR #129710 — [mono] AOT-compile marshalling wrappers for layout classes with class-typed fields**
    クラス型フィールドを持つレイアウトクラスのマーシャリングラッパーを AOT コンパイル対応に変更。Mono AOT のカバレッジ拡大。

29. **PR #129708 — [mono] AOT-compile methods that use the IL jmp opcode**
    IL の `jmp` オペコードを使用するメソッドを AOT コンパイル対応に。稀なオペコードへの対応拡充。

31. **PR #129701 — Use SearchValues in Utf8JsonReader.SkipWhiteSpace**
    `Utf8JsonReader.SkipWhiteSpace` の実装で `SearchValues` を活用。JSON パース処理のパフォーマンス改善。

33. **PR #129661 — Fix RemoveImpl nullability**
    `RemoveImpl` メソッドの null 許容アノテーションを修正。静的解析の精度向上。

35. **PR #129621 — Enable runtime tests on OpenBSD**
    OpenBSD 上でランタイムテストスイートを実行できるよう対応。OpenBSD ポート作業の一環。

37. **PR #129517 — arm64: Add float/double cast optimisation tests**
    ARM64 向けの float/double キャスト最適化に対するテストを追加。回帰防止と品質保証の強化。

39. **PR #129498 — Fix browser wasm merged test reference discovery**
    ブラウザ WASM でマージテストの参照検出が正しく動作しない問題を修正。

41. **PR #129470 — Port System.Diagnostics.Process to OpenBSD**
    `System.Diagnostics.Process` クラスを OpenBSD に移植。プロセス管理 API が OpenBSD でも利用可能に。

43. **PR #129334 — [wasm][coreclr] Re-enable System.Formats.Nrbf tests**
    WASM + CoreCLR 環境で一時的に無効化されていた `System.Formats.Nrbf` テストを再有効化。

45. **PR #129279 — Add OpenBSD support in System.Net.NetworkInformation**
    `System.Net.NetworkInformation` に OpenBSD サポートを追加。ネットワーク情報 API の OpenBSD 対応。

47. **PR #129043 — Async profiler: V1 (TaskAsync) instrumentation + tests**
    非同期プロファイラーの V1 実装（TaskAsync インストルメンテーションとテスト）。非同期メソッドのパフォーマンス分析基盤が整いました。

49. **PR #129035 — [release/10.0] Reduce contention in UnwindInfoTable during high volume LCG method creation**
    大量の LCG（Lightweight Code Generation）メソッド生成時に `UnwindInfoTable` で発生していた競合を削減。.NET 10.0 へのバックポート。

## microsoft/agent-framework — .NET/Python 両エコシステムで機能強化

Foundry 連携・スキル管理・多様なコネクタ対応が進んでいます。

1. **PR #6714 — .NET: Fix hosted agent crash after tool call by rooting session store under $HOME**
   ツールコール後にホスト型エージェントがクラッシュする問題を修正。セッションストアを `$HOME` 配下に配置することで、プロセス間でのパス解決問題を解消。

3. **PR #6703 — .NET Foundry: add CreateMcpTool projectConnectionId overload**
   `CreateMcpTool` に `projectConnectionId` を受け取るオーバーロードを追加。複数プロジェクトをまたいだ Foundry MCP ツール連携が容易になります。

5. **PR #6694 — Python: Explicitly emit available\_resources and available\_scripts in skill content**
   スキルコンテンツに `available_resources` と `available_scripts` を明示的に出力するよう変更。エージェントがスキルの能力を正確に把握できるようになります。

7. **PR #6693 — .NET/Python: Purview: prefer token principal for user identity**
   Microsoft Purview 連携でユーザー識別にトークンプリンシパルを優先するよう変更。認証の一貫性と安全性を向上。

9. **PR #6686 — .NET: Fix SearchDirectoriesForSkills to stop recursing after finding SKILL.md**
   `SKILL.md` を発見した時点で深い再帰探索を停止するよう修正。スキル検索の効率化とパス衝突防止。

11. **PR #6680 — .NET: Add IncludeDetailedErrors option for skill script execution**
    スキルスクリプト実行に `IncludeDetailedErrors` オプションを追加。エラー発生時の詳細情報が取得できるようになり、デバッグが格段に楽になります。

13. **PR #6665 — Python: track dependency maintenance PR creation**
    依存関係のメンテナンス PR 作成をトラッキングする機能を追加。ライブラリ更新サイクルの可視化。

15. **PR #6640 — Python: surface cache and reasoning token counts for Bedrock and Gemini connectors**
    Amazon Bedrock および Google Gemini コネクタで、キャッシュトークン数と推論トークン数を公開。コスト分析や最適化に役立ちます。

17. **PR #6631 — .NET: [BREAKING] Support archive-type skills in AgentMcpSkillsSource**
    `AgentMcpSkillsSource` でアーカイブ型スキル（`.zip` 等）をサポート。**破壊的変更**を含むため、既存ユーザーは移行ガイドを確認してください。

19. **PR #6630 — docs: Add Python session identity ADR**
    Python におけるセッション識別の設計判断を記録した ADR（Architecture Decision Record）を追加。意思決定の背景と根拠がドキュメント化されました。

21. **PR #6557 — .NET samples: migrate coding samples to Foundry-first AIProjectClient**
    .NET コーディングサンプルを Foundry-first の `AIProjectClient` ベースに移行。最新の推奨パターンでサンプルが刷新されました。
