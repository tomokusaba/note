## dotnet/runtime

.NET ランタイムでは、JIT の最適化改善や、Wasm・AOT・テスト周りの強化が中心でした。

### JIT / コンパイラ改善

1. **[#130186] JIT: initialize emitCurIG at declaration**
   RISC-V で emitCurIG の未初期化に起因するクラッシュを防ぐ修正です。低レベルの安定性を高める内容として確認しておく価値があります。
   [PR を見る](https://github.com/dotnet/runtime/pull/130186)

3. **[#130181] JIT: Enable tail await optimization in async versions in tier 0**
   async 版 tier 0 で tail await 最適化を有効化し、非同期コードの実行効率を改善しています。パフォーマンスに関心のある実装では見逃せない変更です。
   [PR を見る](https://github.com/dotnet/runtime/pull/130181)

### WebAssembly / プラットフォーム

1. **[#130153] [Wasm/RyuJit] treat all try side-entries as try entries for throw helpers**
   Wasm の例外経路で try side-entry を throw helper として扱うようにし、例外の制御フローを安定化しています。実行時の挙動をより予測しやすくする修正です。
   [PR を見る](https://github.com/dotnet/runtime/pull/130153)

3. **[#129762] Enable ICF on Linux**
   Linux でも ICF を有効化し、バイナリサイズの圧縮と最適化適用範囲を広げています。配布サイズや実行ファイル最適化に関心がある環境で意味があります。
   [PR を見る](https://github.com/dotnet/runtime/pull/129762)

### API / 互換性

1. **[#129624] Added async overload of ChangeToken.OnChange**
   ChangeToken.OnChange に async overload を追加する変更で、破壊的変更として既存の async lambda が別オーバーロードに結びつくため、呼び出し側の挙動が変わる点に注意が必要です。
   [PR を見る](https://github.com/dotnet/runtime/pull/129624)

### テスト / 静的解析

1. **[#130132] Enable trim/AOT analyzers on System.Text.Json source generator tests**
   System.Text.Json の source generator テストに trim / AOT analyzer を追加し、静的解析のカバレッジを強化しています。生成コードの品質向上につながる変更です。
   [PR を見る](https://github.com/dotnet/runtime/pull/130132)
