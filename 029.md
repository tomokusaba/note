# 2026年6月12日分：Visual Studio と Azure DevOps と Windows App CLI の更新

## 先に結論

今日は、**Visual Studio の PR レビュー強化**、**Azure DevOps 向け Copilot Autofix**、**Windows App CLI の更新** がまとまって出てきました。  
既存の開発フローをそのまま少し便利にする、実務寄りの更新が中心です。

## Developer Blogs（.NET 目線で読む）

### Visual Studio で PR レビューが完結する

- [Review pull requests without leaving Visual Studio](https://devblogs.microsoft.com/visualstudio/review-pull-requests-without-leaving-visual-studio/)

Visual Studio から、GitHub と Azure DevOps の pull request を **開く・確認する・コメントする・承認する** ところまでできるようになりました。  
ブラウザに切り替えずにレビューを進められるので、日常的に PR を触る人にはかなり効きます。

特に便利なのは次のあたりです。

1. ブランチを checkout しなくてもレビューできる  
2. コメント、返信、承認まで IDE で完結する  
3. Copilot がコメント内容から修正案を作れる

### Azure DevOps でも Copilot Autofix が使える

- [Copilot Autofix for GitHub Advanced Security for Azure DevOps](https://devblogs.microsoft.com/devops/copilot-autofix-for-github-advanced-security-for-azure-devops/)

Azure DevOps を使っているチーム向けに、Copilot Autofix が **limited private preview** で登場しました。  
CodeQL が見つけたアラートに対して、修正案をその場で生成して pull request まで持っていけます。

これは、**検出から修正までの往復を短くする** 更新です。  
既存の Azure Repos から GitHub にすぐ移れない組織でも、セキュリティ修正の初速を上げやすくなります。

### Windows App CLI v0.3.2 は地味だけど実務向き

- [Windows App Development CLI v0.3.2: Bundling support, smarter initialization, and more](https://devblogs.microsoft.com/ifdef-windows/windows-app-development-cli-v0-3-2-bundling-support-smarter-initialization-and-more/)

Windows App Development CLI v0.3.2 では、**MSIX bundle 対応**、`winapp init` の自動検出改善、UI screenshot の品質向上が入りました。  
Windows アプリのパッケージングや初期セットアップを触る人には、細かいけれど使う場面が多い改善です。

特に、複数アーキテクチャをまとめる bundle 対応は、配布や Store 提出の流れにそのまま効きます。

## GitHub Changelog（既存ユーザーへの影響が大きいもの）

### Agentic Workflows が PAT なしで使えるようになった

- [Agentic workflows no longer need a personal access token](https://github.blog/changelog/2026-06-11-agentic-workflows-no-longer-need-a-personal-access-token/)

GitHub Actions の `GITHUB_TOKEN` を使って、agentic workflows を動かせるようになりました。  
これで、長寿命の PAT を作成・保管する運用が不要になり、**セキュリティと管理負荷の両方が軽くなります**。

組織所有のリポジトリで使う場合は、AI credits の課金先が組織側になります。  
コストセンターや token usage の管理も合わせて見直すと、運用しやすくなります。

### GitHub Agentic Workflows が public preview になった

- [GitHub Agentic Workflows is now in public preview](https://github.blog/changelog/2026-06-11-github-agentic-workflows-is-now-in-public-preview/)

agentic workflows は、issue triage や CI failure analysis、ドキュメント更新のような反復作業を自動化するための仕組みです。  
Markdown で定義したワークフローを Actions YAML にコンパイルするので、既存の runner groups や policy をそのまま活かせます。

安全面も意識されていて、**read-only 既定**、sandboxed container、safe outputs、threat detection まで一通り揃っています。  
自動化を増やしたいけれど、ガードレールは崩したくない、という既存ユーザーに向いた更新です。

### Copilot CLI に `/settings` が入った

- [Copilot CLI: configure everything from one place with /settings](https://github.blog/changelog/2026-06-11-copilot-cli-configure-everything-from-one-place-with-settings/)

Copilot CLI の設定が、`/settings` でひとまとめになりました。  
散らばっていた設定コマンドや設定ファイル編集を減らせるので、**毎日 CLI を触る人ほど効く改善** です。

## まとめ

- **Visual Studio** で PR レビューが IDE 完結になった
- **Azure DevOps** でも Copilot Autofix が見えてきた
- **Windows App CLI** は、配布と初期セットアップの手間を減らした
- **GitHub の agentic workflows** は、PAT 依存を減らして自動化を進めやすくした
- **Copilot CLI の `/settings`** で、日々の設定変更がかなり楽になった
