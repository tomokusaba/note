### WinApp CLI で .NET アプリにパッケージアイデンティティを付与する

**[Packaging and Package Identity for .NET apps with WinApp CLI on Windows](https://devblogs.microsoft.com/dotnet/packaging-dotnet-apps-winapp/)**（Zachary Teutsch）

.NET デスクトップアプリ（WPF・コンソールなど）に Windows のパッケージアイデンティティを手軽に付与できる **WinApp CLI** を紹介した記事です。

パッケージアイデンティティとは、Windows OS 内でアプリを「登録済み」として扱うための一意識別子で、これがないとプッシュ通知・Windows AI API・バックグラウンドタスク・ファイルハンドラーといった Windows 限定機能が利用できません。従来は appxmanifest やビルド設定の手作業が必要でしたが、WinApp CLI を使うと次の 2〜3 ステップで完結します。

1. `winapp init . --use-defaults` でプロジェクトを初期化（`Package.appxmanifest` 生成・必要な NuGet 参照追加を自動化）
2. 通常どおり `dotnet run` を実行するとパッケージアイデンティティ付きで起動
3. 配布用 MSIX は `winapp pack` で作成

WinGet でインストール可能です：

```powershell
winget install Microsoft.winappcli --source winget
```

**[Claude Opus 4.8 (fast mode) is now in preview for GitHub Copilot](https://github.blog/changelog/2026-06-29-claude-opus-4-8-fast-mode-is-now-in-preview-for-github-copilot)**

### Issue 作成権限をコラボレーターのみに制限できる設定が追加

**[Restrict issue creation to collaborators only](https://github.blog/changelog/2026-06-29-restrict-issue-creation-to-collaborators-only)**

リポジトリ管理者が Issue の作成を書き込み権限を持つコラボレーターのみに制限できるようになりました。スパム的な Issue の増加を抑えたいオープンソースプロジェクトや企業リポジトリに有用です。

設定方法：リポジトリ **Settings → Features → Issues** にある「Creation allowed by: Collaborators only」を選択するだけです。制限が有効な場合、**Issues・Comments・Discussions・Projects・Copilot** のすべてのエントリポイントから、書き込み権限のないユーザーは Issue を作成できなくなります。PR 作成制限と同様の粒度でコントロールできるようになった形です。
