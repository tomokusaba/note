## dotnet/runtime

runtime では、JIT や GC の正しさに関わる修正が中心でした。Wasm や CI の改善も含まれています。

1. **[#130352] JIT: fix GC holes in large stack-target struct block ops**
   大きなスタック対象構造体のコピーやゼロ初期化で GC ポインタの torn state が見える可能性があったため、GC-aware な経路に戻す修正が入りました。JIT と GC の相互作用に関わる重要な正しさ改善です。
   [PR を見る](https://github.com/dotnet/runtime/pull/130352)

3. **[#130317] JIT: Make all static base helpers byref typed in async functions**
   非同期関数内での静的ベースヘルパーの扱いを byref 型として統一し、CSE の誤りを防ぐ修正です。async code path の最適化安定性を高めています。
   [PR を見る](https://github.com/dotnet/runtime/pull/130317)

5. **[#130267] [Wasm] handle non-catch try entries in SCC switch dispatch**
   WebAssembly で例外制御の dispatch が非 catch try エントリでも正しく扱えるようになり、プラットフォーム固有の挙動が改善されました。
   [PR を見る](https://github.com/dotnet/runtime/pull/130267)

7. **[#130264] Migrate devdiv/dotnet-core-internal-tooling to WIF service connection**
   内部ツールの CI で使うサービス接続が WIF ベースへ移行され、認証管理の運用が整理されました。ユーザー向けの機能変更ではありませんが、ビルド基盤の安定性を支える重要な更新です。
   [PR を見る](https://github.com/dotnet/runtime/pull/130264)
