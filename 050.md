### [Your agent already has a plan](https://devblogs.microsoft.com/blog/your-agent-already-has-a-plan)

*Garry Trinder 著 / 2026-06-26 (UTC)*

AI コーディングエージェントは、ドキュメントを読む前にすでに「計画」を立てている——そのメカニズムと、ドキュメント設計で正しい動作を引き出す方法を解説したブログです。.NET ツールや SDK のドキュメントを書いている開発者・テクニカルライターに直接刺さる内容です。

**主なポイント：**

2. エージェントは**タスクを受け取った瞬間にトレーニングデータをもとに計画を立て**、そのあとドキュメントを読む。ページを取得しても「既存の計画を補強するか否か」という目線で読み流されるため、Tips や推奨情報は計画に反しない限り無視される。
4. 解決策は「正しいやり方を詳しく説明する」ではなく、**「エージェントの想定する手順が失敗することを明示的に宣言する」**こと。「`package.json` を直接編集するだけではビルドエラーになる」のように計画を無効化する断言がなければ、エージェントは計画を変えない。
6. SPFx プロジェクトのアップグレードを例に、警告文で「既定の手順は失敗する」と明記したところ **5/5 回** で正しいツール（CLI for Microsoft 365）を使用。Tips のみでは 1/5 回にとどまった。
8. 実践手順は「①エージェントのデフォルト計画を観察 → ②その計画が失敗することをページ上で明示 → ③直後に正しい代替手段を提示」の 3 ステップ。

### [GitHub Desktop 3.6: Worktrees and deeper Copilot integration](https://github.blog/changelog/2026-06-26-github-desktop-3-6-worktrees-and-deeper-copilot-integration)

*2026-06-26 (UTC)*

**GitHub Desktop 3.6** がリリースされ、Git ワークツリー対応と Copilot SDK への移行という大型アップデートが届きました。

2. **Git ワークツリーのネイティブサポート**：複数ブランチを並行して操作可能になりました。コーディングエージェントが独立したワークツリーセッションを使う場面に自然にフィットします。
4. **Copilot SDK への移行**：コミットメッセージ生成とマージコンフリクト解消が Copilot SDK 上で動作するようになり、どちらの機能にも**モデルピッカー**が追加。利用可能なモデルから選択できます。また BYOK（Bring Your Own Key）でサードパーティプロバイダーやローカルモデルへの接続も可能。
6. コミットメッセージ生成は `.github/copilot-instructions.md` や `AGENTS.md` のカスタム指示に対応し、リポジトリのコミットメタデータルールも遵守するようになりました。
8. マージコンフリクト時に Copilot が**変更内容の説明と解決案を提示**し、ユーザーが確認・編集してからマージを完了できるようになりました。

macOS・Windows 向けに公開済み。[github.com/apps/desktop](https://github.com/apps/desktop) からダウンロードできます。
