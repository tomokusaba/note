## GitHub Changelog

### GitHub Issues のエージェント自動化に承認・確信度・理由を追加

[GitHub Issues のエージェント自動化](https://github.blog/changelog/2026-07-23-agent-automation-controls-in-github-issues-in-public-preview/)がパブリックプレビューになりました。ラベル、フィールド、Issue type、クローズ、担当者の変更について、エージェントが変更理由と確信度を提示し、必要に応じて提案として保留できます。

リポジトリ管理者は、自動適用する確信度のしきい値も設定できます。Issue の自動トリアージを使うチームでは、速度を維持しながら確認が必要な変更だけをレビューする運用を作りやすくなります。なお、承認はセキュリティ境界ではない点には注意が必要です。

### Projects と Issue fields で複数選択が可能に

[Projects と Issue fields の複数選択フィールド](https://github.blog/changelog/2026-07-23-multi-select-fields-for-projects-and-issues-in-public-preview/)がパブリックプレビューになりました。ひとつの項目に複数のチーム、モジュール、カテゴリを設定でき、サイドバーから値を選択して Project のビューで絞り込めます。

従来はひとつに絞っていた分類をそのまま管理できるため、横断チームや複数コンポーネントにまたがる Issue を扱う Project で特に有用です。

### GitHub Mobile から失敗した Actions を Copilot に修正依頼

[GitHub Mobile の「Fix with Copilot」](https://github.blog/changelog/2026-07-23-github-mobile-fix-failing-actions-checks-with-copilot-cloud-agent/)により、プルリクエストで失敗した GitHub Actions のチェックから Copilot cloud agent に調査と修正を依頼できるようになりました。Copilot は既存のプルリクエストの上に新しいプルリクエストを作成し、修正案をレビューに回します。

最新の iOS / Android 版 GitHub Mobile で利用できます。外出先でも失敗した CI の一次対応を始められるため、レビュー待ちのプルリクエストを滞留させにくくなります。

### GitHub MCP Server が次の MCP 仕様をサポート

[GitHub MCP Server](https://github.blog/changelog/2026-07-23-github-mcp-server-supports-the-next-mcp-specification/)が、ステートレス化された MCP の次期仕様をサポートしました。セッションと `initialize` がなくなり、クライアントは並列にハンドシェイクできます。

GitHub は Redis セッションの廃止、リクエスト本文の深い検査の回避、elicitation 実装の更新を行っています。Tier 1 SDK は後方互換性を維持しているため、既存の利用者に直ちに必要な対応はありません。

### Linear で Copilot cloud agent が一般提供に

[Linear 向け Copilot cloud agent](https://github.blog/changelog/2026-07-23-copilot-cloud-agent-for-linear-is-now-generally-available/)が一般提供になりました。Linear の Issue を Copilot に割り当てると、内容を解析してドラフトのプルリクエストを作成し、進捗を Linear のアクティビティに共有します。

Linear 側からモデル、リポジトリ内のカスタムエージェント、ベースブランチと作業ブランチを選び、コメントで作業中のセッションを指示できます。Linear を開発管理に使う Copilot Pro、Pro+、Business、Enterprise の利用者にとって、Issue から実装までの流れをつなげる選択肢になります。
