# 自宅サーバーを始めたのは、卒業研究がきっかけだった

先に結論を書くと、私が自宅にサーバーを置くようになったのは、**卒業研究でサーバーを触ったことが直接のきっかけ**です。

そこで得た経験が「家でも動かしてみたい」という気持ちに変わり、インターネット定額接続やDDNSの普及という追い風が重なって、気がついたら自宅サーバーを持つことが当たり前になっていました。

## 卒業研究でサーバーを触った

大学の卒業研究で、個人に最適化した学習教材を提供するシステムを作りました。「繰り返し学習と実際の習熟度の相関を調べる」というのが研究テーマで、そのシステムをWebで動かすために選んだのが、Windows 2000 Server・IIS・ASP（当時はまだ「クラシックASP」とは呼ばず、単にASPでした）・SQL Server 2000という構成です。

キャンパスのイントラネットを基本としながら、一部はインターネットにも公開する形で運用していました。  
この「外に向けて公開している」という感覚は、思いのほか大きかったです。IISの設定を調整したり、ネットワークを意識しながら運用したりする経験が、そのまま実地の勉強になりました。

基本的なネットワークの知識と、IISを動かすことへの自信は、ここで身についたと思っています。

## レンタルサーバーではできないことがあった

当時、レンタルサーバーはCGI/Perl中心の時代でした。  
wikiを動かすにはPHPが必要で、PukiWikiのようなツールを使おうとすると、PHP対応のサーバーをわざわざ探さなければなりませんでした。選択肢は限られていて、PHP対応だったとしても、アクセスが増えてくるとサーバー負荷の問題で制限がかかることもありました。

自宅サーバーなら、サーバーサイドのプログラムを自由に動かせます。PHPはもちろん、データベースも自分で用意できます。この自由さが、自宅サーバーの大きな魅力でした。

誰かの都合に合わせなくてよい、というのは、思った以上に気持ちのいいものでした。

## フラット接続とDDNSが重なった

卒業研究を進めていた頃、ちょうどインターネットの定額接続サービスやDDNSが広まりつつありました。

「自宅でも同じことができる」と気づいたとき、卒業研究で慣れた構成をそのまま家でも試してみたくなりました。それが、自宅サーバーの第一歩です。

使っていた技術がIISだったので、自宅サーバーも当然IISで立てました。研究で使った環境を、そのまま家へ延長するような感覚でした。

## 5台の変遷

気づけば、今まで5台のサーバーを使ってきました。

1. 古いノートPC、Windows 2000 Server
2. DELLのサーバー機、Windows 2000 Server
3. DELLのサーバー機、Windows Server 2008
4. 富士通のサーバー機、Windows Server 2016
5. MS-01、Windows Server 2025

最初の1台を除き、ある時期以外はずっと実家に置いています。  
東京に引っ越したあとも、実家にサーバーを置く体制は変わっていません。4台目からは「東京に住みながら実家のサーバーを維持している」という形です。

この遠隔運用を成立させているのが、今のVPN構成です。

## YAMAHA RTXとVPNで遠隔運用する

東京の自宅と実家の両方にYAMAHAのRTXシリーズのルーターを入れて、VPNを張っています。  
これによって、物理的に離れた場所にあるサーバーを、ネットワーク的には同じ場所のように扱えます。

ルーターを自分で設置してVPNを構成する、というのも、卒業研究から続く「自分で動かしてみる」という感覚の延長線上にある話だと思っています。

---

卒業研究でサーバーを触らなければ、自宅サーバーを始めようとは思わなかったかもしれません。  
「研究のためにサーバーを動かした」という体験が、「自分でサーバーを持ち続けてもいい」という感覚に変わっていきました。

5台目のMS-01にWindows Server 2025を入れた今も、その感覚は変わっていないと思います。  
学習のために使ったツールが、いつの間にか長く続く趣味になっていた。振り返ってみると、そんな経緯でした。
