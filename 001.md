# Microsoft MVP の推薦で、私が大事にしていること

結論から書くと、私が Microsoft MVP の推薦で一番大事にしているのは、**自分より優れた技術スタックを持っているかどうか**です。

これは「自分が評価できる範囲に収まっている人を推薦する」という意味ではありません。むしろ逆で、自分よりも深い専門性や、自分にはない視点を持っている人だからこそ、推薦したいと思います。

今まで 5 人の推薦をして、いずれも Microsoft MVP になっています。

その方々に共通しているのは、やはり自分より優れた技術スタックを持っていたことでした。

## 推薦文章が書けるかどうか

なぜそこを重視するのか。

ひとつは、自分より優れていない場合、推薦文章が書きにくいからです。

推薦は、単に「この人は良い人です」と書くものではありません。その人がどのような活動をしていて、どこに強みがあり、なぜ Microsoft MVP としてふさわしいのかを言葉にする必要があります。

そのときに、推薦する側が「この人は自分よりも明らかにすごい」と思える要素があると、文章に説得力が出ます。

技術的な深さでも、OSS への貢献でも、コミュニティへの向き合い方でも、Microsoft 以外の技術を理解したうえでフィードバックできる力でもよいと思います。

他にはない強みがある人は、推薦する側としても自然に言葉が出てきます。

## Microsoft MVP になることはゴールではない

もうひとつ大事なのは、Microsoft MVP になること自体がゴールではないということです。

受賞はとても大きな節目です。ただ、本当に求められるのは、その先で何をなすかだと思っています。

Microsoft MVP になったことをきっかけに、どのような発信を続けるのか。どのようにコミュニティに還元するのか。どのように技術を広げていくのか。

そこまで考えると、やはりある程度の技術力は必要なのではないかと思います。

技術力があるから、人に伝えられる。

自分の言葉で語れるから、周りの人に届く。

そして、届いたものが次の活動につながっていく。

そういう継続的な流れを作れる人を推薦したいです。

## 推薦したいと思う人たち

これまで推薦した方々は、それぞれに素晴らしい技術者でした。

優れた OSS を開発している人もいました。

Microsoft 以外の技術にも精通していて、その知見を Microsoft の技術やコミュニティにフィードバックできる人もいました。

特定の分野に深く入り込み、他の人にはない視点で発信できる人もいました。

共通していたのは、単に詳しいだけではなく、その知識や経験を人に伝えられることです。

技術を持っているだけでなく、それを共有し、周囲に良い影響を与えられる。

そういう人が Microsoft MVP になることで、コミュニティ全体にも良い循環が生まれると思っています。

## 新規受賞者を増やすことの意味

コミュニティ活動を続けていくうえで、Microsoft MVP の新規受賞者を増やすことは重要なことです。

もちろん、数だけを増やせばよいという話ではありません。

一方で、候補者を探すこと自体が、コミュニティを広げる活動でもあります。

まだ表に出きっていない素晴らしい技術者を見つける。活動を続けている人の価値を言葉にする。次の一歩を後押しする。

そういう積み重ねが、コミュニティの厚みにつながると思います。

だからこそ、品質も量もどちらも大事です。

推薦する側には、その両方を見極める責任があります。

## まとめ

私にとって Microsoft MVP の推薦は、単なる手続きではありません。

その人の活動を見て、強みを言葉にし、これからの継続的な活動につながるように後押しすることです。

だから私は、推薦するときに「自分より優れた技術スタックを持っているかどうか」を大事にしています。

Microsoft MVP になることをゴールにするのではなく、それをきっかけに何をなすのか。

そこに向き合える人を見つけ、推薦し、継続的な活動につながるようにコミットしていきたいです。
