## microsoft/aspire

Aspire では、CLI／開発ツールの運用改善に加え、Radius と Foundry Hosted Agent という新しい連携面が大きく動いています。

### ツール・CI・生成コード

1. **[#18723] Make bundled dotnet-inspect skill a thin bootstrapper that defers to the tool**
   Aspire CLI に同梱していた `dotnet-inspect` の詳細ガイドを薄い導線に置き換え、インストール済みツールが提供するバージョンに対応したガイドへ委ねました。リリースごとに埋め込みドキュメントが古くなる問題を避け、コマンドやフラグの追随性を高めます。
   [PR を見る](https://github.com/microsoft/aspire/pull/18723?WT.mc_id=DT-MVP-5004827)

3. **[#18717] fix: batch sdk dump in generate-ats-diffs to fix disk space exhaustion**
   定期ワークフローで約56個の統合プロジェクトごとに `sdk dump` を実行していたため、AppHost の生成物が蓄積してディスクを使い切る問題を修正しました。1回のバッチ実行で出力を作り、各プロジェクトへ振り分ける方式に変えています。
   [PR を見る](https://github.com/microsoft/aspire/pull/18717?WT.mc_id=DT-MVP-5004827)

5. **[#18705] Update Spectre.Console to 0.57.2**
   Spectre.Console を 0.57.0 から 0.57.2 へバージョンアップしました。長いファイルパスがコンソール上で不自然に折り返される問題に対し、狭い幅での回帰テストも追加しています。
   [PR を見る](https://github.com/microsoft/aspire/pull/18705?WT.mc_id=DT-MVP-5004827)

7. **[#18698] Flatten single optional DTO 'options' parameter in Go polyglot generator**
   C# 側で `options` だけが任意 DTO の場合、Go 生成コードでは余分なオプション用構造体を作らず、その DTO を直接渡せるようにしました。RPC ペイロードは変えずに、TypeScript 側に近い呼び出し形へ整えています。
   [PR を見る](https://github.com/microsoft/aspire/pull/18698?WT.mc_id=DT-MVP-5004827)

### ダッシュボードと統合

1. **[#18701] Fix trace 'not contains' filter not excluding matching traces**
   トレースの否定フィルターが、複数の span のうち一つでも条件を満たすと誤って残る問題を修正しました。`NotContains` と `NotEqual` は、条件に反する span が一つでもあればトレースを除外する判定に切り替わります。
   [PR を見る](https://github.com/microsoft/aspire/pull/18701?WT.mc_id=DT-MVP-5004827)

3. **[#18696] Add Aspire.Hosting.Radius: Radius as a compute environment (+ cloud providers)**
   プレビュー統合 `Aspire.Hosting.Radius` を追加し、AppHost から Radius をコンピュート環境として扱えるようにしました。ローカル実行のリソースグラフを保ったまま、publish／deploy 時には Radius への展開へつなげ、Azure・AWS の資格情報もコード側で構成できます。
   [PR を見る](https://github.com/microsoft/aspire/pull/18696?WT.mc_id=DT-MVP-5004827)

5. **[#18692] Update Foundry hosted agent protocol payload**
   **破壊的変更**として、Foundry Hosted Agent のデプロイ要求を現行スキーマに合わせ、プロトコル指定とコンテナーイメージの配置を更新しました。`ContainerProtocolVersions` は `ProtocolVersions` に改名され、TypeScript・Java・Go の `asHostedAgent` 呼び出しもプロトコルとバージョンを必須の位置指定引数で渡す必要があります。
   [PR を見る](https://github.com/microsoft/aspire/pull/18692?WT.mc_id=DT-MVP-5004827)

## dotnet/runtime

dotnet/runtime は 19 件です。cDAC／デバッガーの整備、モバイル・WASM などのプラットフォーム対応、ライブラリの性能・テスト、CI／リリース修正に分けて見ます。

### cDAC・デバッガー

1. **[#130501] Fix cDAC big-RVA field offset detection to mask packed type bits**
   cDAC が大きな RVA のフィールドオフセットをメタデータから解決できなかった不具合を修正しました。型ビットを含む値をそのまま比較せず、オフセット部分をマスクしてから判定することで、従来は到達しなかった分岐を機能させています。
   [PR を見る](https://github.com/dotnet/runtime/pull/130501)

3. **[#130448] [cDAC] Remove vestigial JupiterRefCount from HandleData contract**
   既に意味のある値を持たない `JupiterRefCount` を、内部の managed cDAC 契約から取り除きました。一方で公開済みの `SOSHandleData` COM ABI のフィールドは残し、互換性のため値は引き続き `0` として返します。
   [PR を見る](https://github.com/dotnet/runtime/pull/130448)

5. **[#130275] Fix x86 Debugger_STRData layout and ESP adjustment in cDAC DBI**
   x86 の DBI でスタックフレームと継続情報に使うアドレス表現を固定幅の `CORDB_ADDRESS` にそろえました。32ビット DBI と cDAC の互換性を改善し、不要になった ESP 調整の仕組みも整理しています。
   [PR を見る](https://github.com/dotnet/runtime/pull/130275)

7. **[#129267] [cDAC] Implement GetNativeCodeSequencePointsAndVarInfo for cDAC**
   cDAC の `GetNativeCodeSequencePointsAndVarInfo` を実装し、ネイティブのアロケーターに依存しないコールバック形式へ改めました。メタデータトークンを持たない非同期メソッドなどで署名解決が失敗する箇所にも、補助 API を追加しています。
   [PR を見る](https://github.com/dotnet/runtime/pull/129267)

### プラットフォーム・ホスト

1. **[#130461] Remove outdated comment in OSArchitecture**
   `OSArchitecture` のコメントを、.NET 7 以降の ARM64 上での x64 エミュレーションの実態に合わせて更新しました。動作変更ではなく、現在の互換性仕様と説明のずれを解消するメンテナンスです。
   [PR を見る](https://github.com/dotnet/runtime/pull/130461)

3. **[#130402] [clr-ios] Default Apple mobile CoreCLR ReadyToRun to composite regardless of BuildTestsOnHelix**
   Apple モバイル向け CoreCLR ReadyToRun で、Helix 上のテストビルド時だけ composite Mach-O の既定値が外れる問題を修正しました。ビルド条件にかかわらず、他の Apple モバイルイメージと同じ複合形式を既定にします。
   [PR を見る](https://github.com/dotnet/runtime/pull/130402)

5. **[#130396] Use `DOTNET_HOST_` instead of `COREHOST_` in docs**
   ドキュメント中のホスト環境変数を、.NET 10 で主となる `DOTNET_HOST_` 名へ更新しました。従来の `COREHOST_` はフォールバックとして扱われることも、あわせて明示しています。
   [PR を見る](https://github.com/dotnet/runtime/pull/130396)

7. **[#130340] Fix "GMT Standard Time" incorrectly interpreted as fixed-offset zone on Android**
   Android で `GMT Standard Time` を固定オフセットの GMT と誤認し、夏時間なしの時刻帯として返す問題を修正しました。`GMT+5` のような数値オフセットだけを特別扱いし、英国時間は IANA ルックアップへ正しく進むようにしています。
   [PR を見る](https://github.com/dotnet/runtime/pull/130340)

9. **[#130168] [mobile] Disable NativeAOT-specific test failures on Apple mobile**
   Apple モバイルの NativeAOT でのみ失敗するテストを無効化しました。キーチェーンの権限、リフレクション、設定ファイル、接続終了時の HTTP といった NativeAOT 固有の失敗に限定し、CoreCLR と Mono で通るテストは継続します。
   [PR を見る](https://github.com/dotnet/runtime/pull/130168)

11. **[#130142] [browser] atob polyfil for feature detection in V8**
   V8 での機能検出に向けて `atob` のポリフィルを扱う変更です。PR 本文が空のため、具体的な適用範囲や挙動はタイトルから確認できる範囲に限定します。
